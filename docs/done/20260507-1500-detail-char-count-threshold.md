# 任务详情字数计数 + 长度提示 — 开发计划

> 对应需求（来自 docs/TODO.md）：
> 任务详情字数计数 + 长度提示：编辑器现已显字数；超过 2000 字时字色转琥珀，提示笔记可能太长，考虑拆分到子任务。

## 目标

detail.md 编辑器现在显示 "N 字" 灰字。但用户写得越来越长（几千字）时没
有视觉提示笔记规模。本轮加阈值：超过 2000 code point 时字数转琥珀色 +
title 提示考虑拆分子任务。

## 非目标

- 不强制限制长度 —— detail.md 没有 hard cap；琥珀色仅是提醒，不阻拦保存。
- 不分级 (warn / error 多档) —— 单一阈值简化判断；用户感知到颜色变化已
  足够。
- 不 emit 弹层 / 通知 —— 视觉变色已是合适的轻量提示。

## 设计

阈值常量 `DETAIL_LENGTH_HINT_THRESHOLD = 2000`：选 2000 是因为典型一页
A4 中文 ~1000 字，2 页 ≈ 笔记开始臃肿的临界点；保留主观阈值不深究。

颜色：琥珀 `#b45309`（与 task soon-due 同色系，"提醒但非紧急"）。

`title` hover 文案：超过阈值时换成 "笔记已超 2000 字，考虑拆出子任务避免
单段过长 / 难浏览"。

## 阶段划分

| 阶段 | 范围 |
| --- | --- |
| **M1** | 阈值常量 + 字数显示分支（颜色 + title） |
| **M2** | tsc + build + cleanup |
