# LlmLogView 多 chip 过滤

## 需求

LLM 日志 tab 全量列出窗口内 entries（已有"加载更早"增量加载），但用户想
看"今天用 gpt-4o 调过的所有 round"或"调过 web_search 的"，得肉眼扫 + ⌘F。
加 model / tool 两组 chip 过滤，OR-within / AND-between 语义。

## 实现

`src/components/panel/LlmLogView.tsx`：

- 新 state：
  - `modelFilter: Set<string>` —— 选中的 model 名集合（空 = 不过滤）
  - `toolFilter: Set<string>` —— 选中的 tool 名集合（空 = 不过滤）
- `distinctModels / distinctTools` useMemo：从 `entries` 派生频次降序、
  字典序破平的 (name, count) 列表。tool 维度从 `response.tool_calls` 全集
  统计，一条 entry 多 tool 各计一次
- `visibleEntries` useMemo：两维度都过 → 显；任一维度空集 = 不过滤该维度；
  非空 = OR-within（任一命中即通过）。两个维度 AND-between
- `toggleSet` 通用 helper：set 切换值
- Toolbar 升级：
  - 顶行：刷新 + (条件)"清过滤" + 计数 "显示 X / 全集 Y / 窗口 Z"
  - 新增 chip 行：model 一行 + tool 一行（仅当对应集合非空才渲染）
  - chip 配色：model = sky 蓝；tool = orange，与 LLM 日志摘要 badge 已有
    配色语义对齐
- entries.map 改 visibleEntries.map；新增 "当前过滤无命中" 空态文案 + 提示
- 新 useEffect：modelFilter / toolFilter 变化时 setExpandedIdx(null) —
  visibleEntries 索引重排会让旧 idx 指到无关行，重置让用户自己再点开关注的

## 验证

- `npx tsc --noEmit` clean
- 行为：
  - 入页：toolbar 显 chip 行（前提：window 内有 entries）；点 chip 切红
    底高亮
  - 多 chip 选：model + tool 双过滤，AND 语义生效
  - 全集 50 条里挑 1 个 tool → counter "显示 7 / 全集 50 / 窗口 50"
  - 过滤后无命中 → "当前过滤无命中"占位提示
  - 点 "清过滤" → 两个 Set 都清，回全量视图
  - 2s polling 抓新 entry → distinctModels / distinctTools 自动随之扩展
    （比如出现新 model 名 → 新 chip 自动加入行尾）
  - 切 chip 时若有展开的 round → 自动折叠（避免错位高亮）

## 不在本轮范围

- 没做"按 round 号过滤"：round 是 entry 内部字段，输入精确数字过滤不如
  滚动找直接
- 没做日期范围过滤：log 已按时间倒序展示，目前一窗口最多 200 条，时间维
  度由"加载更早"控制窗口大小
- 没做后端 server-side 过滤：日志文件总大小有限，客户端 derive 性能足够；
  做 server-side 反而要新 IPC 接口
