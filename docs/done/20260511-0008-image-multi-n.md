# /image 支持多张 (-n N)

## 需求

`/image dragon over moon` 一次只能拿一张，要 4 张得敲四遍。OpenAI dall-e-2、SiliconFlow 上的 SD/flux 都原生支持单次多张返回，前端只缺一个 `-n N` 入口。

## 实现

### 后端

`src-tauri/src/commands/image.rs`：

- `image_generate(prompt: String, n: Option<u32>) -> Result<Vec<String>, String>`
- `IMAGE_HARD_MAX_N = 10` 后端硬上限，clamp `n.unwrap_or(1)` 到 [1, 10]
- ImageRequest.n = n_clamped 透传给 `{base_url}/images/generations`
- 各 provider 自己的 n 限制（dall-e-3=1、dall-e-2≤10、SD/flux 多 1-4）由 API enforce，错误透传给前端

注：`n: Option<u32>` 让 invoke 调用方不传 = 缺省 1，旧调用者（如重试按钮）零改动可用。

### 前端

`src/components/panel/slashCommands.ts`：

- `IMAGE_MAX_N = 8` 前端软上限（防误打 -n 100 烧 API quota）
- `kind: "image"` 加 `n: number`
- arg 解析加 `-n N` 标志：正则 `^-n\s+(\d+)\s+(.+)$` 切出 N 与剩余 prompt；不合法（N 越界 / rest 空）整体当 unknown 让用户看错文案
- 没有 -n 标志时 n = 1，行为与之前相同
- 命令 description 改 `生成图：/image <描述>；多张走 /image -n 4 <描述>`

`src/components/panel/PanelChat.tsx`：

- `runImageGenerate(prompt, replaceAtIdx, n: number = 1)`
- pending / 完成 / 失败行的 content 字段在 n>1 时挂上 `（-n N）` 后缀，让用户看到生成方式
- invoke 传 `{ prompt, n }`
- 用户回声 `/image -n N <prompt>`（n=1 时省略 -n）
- 失败时 `imageRetryPrompt` + 新增的 `imageRetryN`（n>1 才写，缺省 1 不污染老数据）一并落到 ChatItem
- 重试按钮 onClick 把 `retryN` 一起传回 → 重试沿用同 N

`src/components/panel/panelChatBits.tsx`：

- `ChatItem.imageRetryN?: number` 配套字段

### 渲染

`CopyableMessage` 已有 `flexWrap: "wrap"`、`max: 160px`、`gap: 6` 的图片网格 —— 多张返回直接平铺，无需新增组件。

## 验证

- `npx tsc --noEmit` clean
- `cargo check` clean
- 行为：
  - `/image dragon` → 单图，与之前一致
  - `/image -n 4 dragon` → 用户回声显 `-n 4`；pending 行显 `（-n 4）`；返回 4 张缩略图平铺
  - `/image -n 100 dragon` → unknown，提示用户错；前端 8 / 后端 10 双层 clamp
  - dall-e-3 model 下 `-n 4` → API 返回 400 → 透传错误 + 🔄 重试按钮（按钮也带 -n 4 重新尝试）
