### `format_idle_7d_reply(rows: &[(title, days_idle, last_update_date)]) -> String`

pure — caller 已 filter pending + updated_at ≥ 7d 前 + sort：

- 空 → 「无 7d+ idle pending — 健康状态」+ 教学指 /touched_thisweek
  （本周活跃 task — sibling scope alt）
- 非空 → header「💤 stale backlog N 条（pending + updated_at ≥ 7 天
  前）」+ 每行 `· 「<title>」 · idle <days> 天（last update YYYY-MM-DD）`
- cap 12（比 /find_speech 系列 8 稍宽 — backlog 列表需要多看几条）
- > 12 时尾「还有 N 条（idle 较短的截掉）」

### Handler (bot.rs)

紧贴 `CatDecay7d`：

```rust
TgCommand::Idle7d => {
    use crate::task_queue::TaskStatus;
    let views = read_tg_chat_task_views(chat_id.0);
    let now_ms = chrono::Local::now().timestamp_millis();
    let cutoff_ms = now_ms - 7 * 24 * 60 * 60 * 1000_i64;
    let mut rows: Vec<(String, i64, String)> = Vec::new();
    for v in &views {
        if v.status != TaskStatus::Pending { continue; }
        let Ok(t) = chrono::DateTime::parse_from_rfc3339(&v.updated_at)
        else { continue };
        let ms = t.timestamp_millis();
        if ms > cutoff_ms { continue; }
        let days = (now_ms - ms) / 86_400_000;
        let last_date = v.updated_at.chars().take(10).collect();
        rows.push((v.title.clone(), days, last_date));
    }
    rows.sort_by(|a, b| b.1.cmp(&a.1).then_with(|| a.0.cmp(&b.0)));
    format_idle_7d_reply(&rows)
}
```

## Key design decisions

- **`pending` only**：与 PanelTasks chip 同语义 — done / cancelled /
  error 不进 inactivity audit；finished 不动是常态非问题
- **snoozed 仍算 idle**：含 [snooze:] marker 的 pending 也列。snooze
  是 owner action 但超 7d 没醒来值得 audit（snooze 时间过头了）
- **7d 阈值与桌面 chip 一致**：避免「桌面显但 TG 不显」错位。也与
  iter #559 PanelTasks 💤 hover chip 同
- **idle 天数 desc + title asc tie-break**：最老 stale 在上；稳定
  输出便测
- **cap 12 vs `/find_speech` cap 8**：backlog audit 用户想多看几条
  （特别是写 weekly report 时按 idle 序列出全部）— 比 keyword 搜结
  果列宽
- **`last_date` 只取前 10 字**：updated_at 是 RFC3339 ISO，前 10 字
  是 `YYYY-MM-DD` ASCII，char-boundary safe + 不依赖额外 parse
- **4 unit tests**：parser + healthy empty fallback + multi-row render
  + cap 12 + tail hint
