# TG `/here_top_cat` 命令（iter #594）— cat-dimension priming

## Background

iter #591-#593 加了 here-* 注入 trio：
- /here_pin: task 维度（pinned）priming
- /here_idle: task 维度（stale）priming
- /here_clear: 撤回

缺 **cat 维度 priming** — 让 pet 知「我当前研究范畴」。本 iter 加
/here_top_cat 把 top 3 cat（按 items 量）作主力 cat context 注入。

## Changes

6-surface 同步（mirror /here_pin）：

1. Enum `HereTopCat`（紧贴 `HereClear`）
2. name / parser / no-args title chain / registry en+zh / help-detail /
   help-table / 三份 drift lists
3. 新 pure `format_here_top_cat_reply(rows: &[(key, count)], until)`
4. Handler scan memory_list + sort desc + take 3 → 拼「📊 主力 cat
   context：cat1 (N1) · cat2 (N2) · cat3 (N3)」 → set_transient_note 60
5. /help_table cat 家族 + family detail map 同步加入

## Output

```
📊 已注入 top 3 cat 到 transient_note（到 HH:MM 失效）

```

空 → 「memory 无 cat — 试 /cat_top 看清单 / /pin <title> 钉 task 先」+
教学指 /here_pin。

## Key design decisions

- **top 3 hard-coded**：与 /audit_summary 5-signal cap 同精神，限输出
  紧凑。owner 想看更多走 /cat_top [N]
- **inline `cat (N)` text**：transient_note 注入紧凑「· cat1 (N1) ·
  cat2 (N2)」一行，让 pet prompt token 经济。reply ack 仍用多行显
  人类可读
- **复用 sort by count desc**：与 iter #589 /cat_top 同 sort 算法。
  本 iter 没单独 helper — 处理小不必抽
- **空时教学指 /cat_top + /pin**：双入口提示让 owner 选择性走（look
  vs 创建）
- **3 unit tests**：parser + 空兜底 + 多 row render
