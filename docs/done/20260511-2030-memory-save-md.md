# PanelMemory 一键导出 .md 文件

## 需求

PanelMemory 顶部已有"📋 导出"按钮，把全部记忆拼成单 markdown 复制到剪贴
板。想做本地备份还得"打开编辑器 → 粘 → 存"，三步。加一个"💾 .md"按钮
直接保存到本地。

## 实现

`src/components/panel/PanelMemory.tsx`：

- 新 `handleExportAllToFile`：
  - 复用 `exportMemoriesAsMarkdown(index)` 生成 markdown 文本
  - `new Blob([md], { type: "text/markdown;charset=utf-8" })`
  - `URL.createObjectURL(blob)` → `<a download="pet-memories-YYYY-MM-DD.md">`
    动态创建 + click + remove
  - 1500ms 后 `URL.revokeObjectURL(url)` —— 同步 revoke 偶发抢在写盘前
    生效让文件空，延迟 release 更稳
  - setMessage 4s 反馈"已保存 N 条记忆到 <filename>"
- 头部按钮组在"📋 导出"旁加"💾 .md"按钮（同 `s.btn` 灰底样式，tooltip
  解释与剪贴板路径的区别 + 文件名规则）

## 为什么不用 Tauri dialog plugin

`@tauri-apps/plugin-dialog` 的 `save()` 走 native save panel 但要：
1. 加 Cargo dep `tauri-plugin-dialog`
2. 注册 plugin 到 `lib.rs`
3. 前端 `import { save }` + 再 `writeTextFile`（需要 fs plugin）

`<a download>` 在 macOS WKWebView 直接调系统 save 面板（与 Chrome 一致行
为；用户的浏览器偏好控制"问保存到哪 / 自动落 Downloads"）。零 dep，与既
有 ImageLightbox 的下载图片走同一条 web 原生路径，行为可预期。

## 验证

- `npx tsc --noEmit` clean
- 行为：
  - 点 💾 .md → 系统 Save 对话框 / Downloads 目录出现
    `pet-memories-2026-05-11.md`
  - 文件内容与 📋 导出一致（markdown 按 category 分段）
  - 重复点击不互盖（同一天 YYYY-MM-DD 同名 → 浏览器自动加 (1) (2) …）
  - index 还没加载（loading）时按钮 disabled
  - 异常路径（罕见：blob 创建失败 / a.click() 抛）→ 红色 message

## 不在本轮范围

- 没做"按 category 分多个文件 + zip"：单 .md 已经包含所有 category（用
  `## category` 段头），用户在编辑器里一目了然；zip 多文件方案要引入
  jszip ~30kb，收益小
- 没做"导出时含 detail.md 全文"：detail 数据多模态（图片 base64 等），
  全量打包会让 .md 体积膨胀；现版只导出 title / description / 元数据，
  适合"快照备份"场景。深度导出留给后续单独命令
- 没改 📋 导出按钮：保留剪贴板路径让"想贴 issue / 聊天"用户少一步
