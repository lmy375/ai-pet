# TG bot 命令描述中英文切换 — 开发计划

> 对应需求（来自 docs/TODO.md）：
> TG bot 默认命令翻译键：现 set_my_commands description 全中文；加 settings.telegram.command_lang `zh` / `en` 切换，让英文用户在 TG 客户端看到英文描述。

## 目标

`tg_command_registry()` 返回的硬编码 5 条命令描述全中文。英文用户在 TG
补全表里看到中文描述需要翻译。本轮加 `settings.telegram.command_lang`
枚举（`zh` 默认 / `en`），按值切换 `merged_command_registry` 的 hardcoded
段描述；custom commands 用户自填，**不**翻译（用户配什么显什么）。

## 非目标

- 不做完整 i18n 框架 —— 只是 5 条命令描述的双语切换；引入 fluent / i18next
  对 5 条字符串过度。
- 不翻译 `format_help_text` 中文文案 —— /help 是详情面板，进 TG 看 /help
  的英文用户场景不广；先做 setMyCommands 的"补全表"翻译这一个高频接触点。
- 不动 `format_command_success` / `format_unknown_command` 等运行时反馈
  —— 这些在 chat-pipeline 边缘，i18n 影响面更大；按需后续再加。
- 不持久化为独立字段类型 —— 用 String enum 序列化（`zh` / `en`），陌生
  值默认 `zh` 兜底（unknown 字符串不让 bot 起不来）。

## 设计

### settings 字段

`TelegramConfig.command_lang: String` default `"zh"`。

### 修改 tg_command_registry

签名：`tg_command_registry(lang: &str) -> Vec<(&'static str, &'static str)>`。
内部 match `lang.to_ascii_lowercase()`：
- `"en"` → 英文条目
- 其它（含 `"zh"`、空、未知）→ 现有中文条目（向后兼容默认）

### 翻译表

```rust
match lang {
    "en" => vec![
        ("task", "Queue a task (!! P5 / !!! P7)"),
        ("tasks", "List tasks dispatched in this chat"),
        ("cancel", "Cancel a task"),
        ("retry", "Retry a failed task"),
        ("help", "Show command help"),
    ],
    _ => vec![ /* current Chinese */ ],
}
```

### 调用方

`merged_command_registry(custom, lang)` 透传 lang 给 hardcoded 部分；
custom 不翻译。

bot.rs 启动时读 `config.command_lang` 传过去。其它（如 typo-suggest 用的
`tg_command_registry()`）可以默认走 zh — 那个仅做 valid name 列表判定，
用不到 description。

### 测试

- `tg_command_registry("zh")` 返回中文（含"队列" / "取消"）
- `tg_command_registry("en")` 返回英文（含 "Queue" / "Cancel"）
- `tg_command_registry("xx")` 兜底 zh
- `tg_command_registry("EN")` 大小写归一
- `merged_command_registry` 透传 lang 后 hardcoded 段对应改变；custom
  描述完全不动（不翻译）

### 前端 settings UI

PanelSettings TG section 加 select：`command_lang` `zh` / `en`，默认 zh。
保存触发 reconnect 才生效（bot 启动期一次性读 settings）。
