# PanelTasks detail.md 编辑面板 > 5000 字 banner

## 需求

任务 detail.md 整篇会被宠物每次"执行该任务"时读入 LLM prompt 上下
文（不分 chunk / 不摘要）。长文不仅烧 token，还让模型抓不到当下要
做的事。当前编辑态 counter 只有一档 muted 色"字符数"，没有把"过长
会影响 pet 表现"传递给用户。

加 > 5000 字阈值显眼 banner + counter 三档配色，给用户清晰的精简
信号。建议性，不阻塞写 / 保存。

## 实现

`src/components/panel/PanelTasks.tsx`，编辑态 toolbar 行 + 新增 banner：

- 字数 counter 改 IIFE，三档配色与阅读态 counter 同语义但更激进：
  - < 2000 字 → muted 色（与现状一致）
  - 2000-5000 字 → amber `#a16207`（warn 不 alarm）
  - > 5000 字 → red `#dc2626` + bold（与 title counter 同档色）
  - 用 `Array.from(editingDetailContent).length` 数 code point（与阅
    读态 counter 同方法，对中文 / emoji 正确）；之前的 `.length` 是
    UTF-16 code unit 数，中文每字 1 单位、surrogate pair emoji 2 单
    位 —— 改 code point 数更直观一致
- toolbar 行下方新增 banner block，仅在 > 5000 字时浮：
  - orange tint 底 + orange tint 字 + 1px 边
  - "⚠ 笔记已超 5000 字 —— detail.md 整篇会被宠物读进 prompt 上下
    文，建议把已完成段落归档或拆子任务，让 LLM 抓重点更稳。"
  - title tooltip 解释"为何 5000 是阈值"
  - 不挡保存按钮，不 disable textarea —— 用户可以继续写

## 验证

- `npx tsc --noEmit`：clean
- 行为：
  - 编辑笔记达 2001 字 → counter 变 amber
  - 编辑笔记达 5001 字 → counter 变 red 加粗 + banner 浮
  - 删字降回 5000 以下 → banner 消失，counter 回 amber
  - banner 不挡保存 → 仍可 ⌘S 写盘
  - 中文 / emoji 计数：100 个汉字 = 100，10 emoji（含 surrogate pair）= 10

## 不在本轮范围

- 没做阈值可配（用户调 5000 → 8000）：5000 是经验值，> 5000 字的
  task detail 在实际使用里几乎都该拆。如果未来用户反馈太严再做配
  置项
- 没做"自动建议拆任务"按钮（点击调 task_split 之类后端）：拆任务
  涉及历史挪移 / 状态切分，不在 banner 一行能完成
- 没改阅读态 counter（仍 > 2000 amber 阈值）：阅读态用户被动看长
  detail 没痛感 + 不会主动加字；编辑态有具体行动空间，激进 5000 阈
  值有意义
- 没做 token 实际估算（按 BPE 估 token 而非 char）：char 是 cheaper
  proxy，5000 字 ≈ 2500-3500 token 在中英文混合下，已足以引导用户
