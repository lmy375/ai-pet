# TG `/here_until <HH:MM>` 命令（iter #601）— transient_note 到期 reset

## Background

iter #591-#595 / #598 加 here-* 注入 + clear + status。owner 注入后想
延长 / 缩短到期时间 — 当前路径：注 60min → 想多 1 小时只能重新调
`/transient <text> 120`（需重抄 text）。不优雅。

本 iter 加 /here_until <HH:MM> — 改当前 transient_note 到期到绝对
HH:MM 时刻；text 保持不变。

## Changes

6-surface 同步：

1. Enum `HereUntil { raw: String }`（紧贴 `HereStatus`）
2. name / parser（含 title chain）/ registry en+zh / help-detail /
   help-table / 三份 drift lists
3. 新 pure `format_here_until_reply(kind, raw, text, target_label,
   minutes)` 4-branch formatter（usage / no_transient / expired / ok）
4. Handler parse_sleep_until_time（复用 iter Sleep helper）+ chrono
   today's HH:MM build + fetch current text + 算 minutes + 调 set_
   transient_note(text, minutes)

## Output

ok:
```
📝 已改 transient_note 到期到 HH:MM（剩 N 分钟）
· <text preview>
```

no_transient:
```
📝 当前无 transient_note — /here_until 仅在已注入时可用。
试 /here_pin / /here_idle / /here_top_cat / /here_recent_done / /transient 注入。
```

expired (target ≤ now):
```
⚠️ 到期已过（target HH:MM，N 分钟前）— transient 立即失效。
· <prior> <text preview>
```

usage:
```
📝⏰ 用法：/here_until <HH:MM>
例：/here_until 16:00 / /here_until 9:30 / /here_until 23
（无效 raw「{raw}」 — 时刻应为 HH:MM 或 H:MM 或 H/HH）
```

## Key design decisions

- **复用 `parse_sleep_until_time`**：与 /sleep_until 共享 HH:MM 解析逻
  辑；接受 `HH:MM` / `H:MM` / `HH` / `H` 4 种形式
- **target ≤ now 时不 set**：expired 警示但不调 set_transient_note —
  避免立即失效破坏当前 transient（owner 误打 9:00 已过；保护 current
  context）
- **text 保持不变**：set_transient_note(text, minutes) 仅改到期；text
  从 get_transient_note() 取既有 — 保 sync
- **4 kind branches in formatter**：usage / no_transient / expired / ok
  — handler 走 if-else 决定 kind 传入
- **target 仅今日 HH:MM（不跨日）**：与 /sleep_until 同语义；若 target
  < now 仅显 expired warning。跨日延长需更高级 API（按需 propose）
- **5 unit tests**：parser（含空 raw）+ 4 kind format（usage / no_
  transient / ok with minutes / expired with negative minutes label）
