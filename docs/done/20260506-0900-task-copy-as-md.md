# 任务详情面板加复制为 Markdown — 开发计划

> 对应需求（来自 docs/TODO.md）：
> 任务详情面板加复制为 Markdown：单任务详情现在只能复制纯文本，加个 "Copy as MD" 输出 `## title\nbody\n— due / pri / tags`，方便贴入笔记。

## 目标

PanelTasks 展开面板里现有两个复制按钮：完整描述 (raw_description) 与进度笔记
(detail_md)。但用户想把任务塞进 Notion / Obsidian / 周记时，要的是一个**结构化
Markdown 片段**：`## title` + 元信息列表 + body + 进度笔记 + 产物。本轮加一个
"Copy as MD" 按钮，一键拼好整段 markdown 写到剪贴板。

## 非目标

- 不动后端 — 全部信息在前端 `TaskView` + `TaskDetail` 已可拿到。
- 不导出 history（butler_history events）— 那是审计流，不是任务文档。导出
  会让 markdown 太长、噪音大。
- 不做 "导出全部任务" — 单任务足够；多任务批导出价值低 + 易膨胀。

## 设计

### Markdown 输出格式

```markdown
## {title}

- **状态**: {status_label}
- **优先级**: P{priority}
- **截止**: {due, 若有}
- **标签**: {#tag1 #tag2 …, 若有}
- **创建**: {created_at}
- **更新**: {updated_at}

{body, 若有}

### 进度笔记

{detail.detail_md, 若有}

### 产物

{result, 若有}
```

可选块（body / detail_md / result / 标签 / 截止）为空时整行/整段省略，避免
出现 "标签：（空）" 这种垃圾行。

### UI

- 在 `s.detailPanel` 顶部（`detailLoading / detailErr / detail` 三态之外的
  外层）插一个 chips 行：「Copy as MD」按钮。
- 复用既有 `handleCopyDetail` 的"按钮变绿 + 1.5s 自动复位"机制，sectionKey
  使用 `${t.title}-asMd`。
- 按钮样式与既有 detail copy 按钮一致（pet-detail-copy-btn class）—— 不增视觉
  分歧。

### 纯函数边界

`formatTaskAsMarkdown(t: TaskView, detail: TaskDetail): string` 放在文件顶部
工具区。这是纯字符串拼装，单元可测；但项目无 vitest，靠 tsc + 手测。
