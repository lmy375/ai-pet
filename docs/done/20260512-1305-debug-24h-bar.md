# PanelDebug 今日 24h 主动开口 mini bar

## 需求

PanelDebug 当前 today / week / lifetime 是单数字 stats，看不出"今天宠物
什么时候说的话最多 / 哪个小时是寂静"。加 24h mini bar 让节奏可视化。

## 实现

### 后端 `src-tauri/src/speech_history.rs`

- 新 pure `hourly_counts_for_date(content, target_date) -> [u64; 24]`：
  - 遍历 speech_history.log 内容行
  - 解 RFC3339 时戳 → 转本地时区 → 比对 target_date → bucket by hour
  - 无效行 / 跨日行跳过
- 新 `get_today_speech_hourly() -> Vec<u64>` Tauri 命令：read_history_content
  + `Local::now().date_naive()` + 调上面 pure helper → 返 24-element Vec
- 注册 `lib.rs`

### 前端 `src/components/panel/PanelDebug.tsx`

- 新 state `hourlyBuckets: number[]`（初值 24 个 0）
- 新 useEffect 挂载时 invoke + 60s 轮询；cleanup 清 interval
- 在既有 "近 N 天" chip 行下方加 mini bar：
  - 左侧 label "今日 24h"
  - 中间 grid 24 列等宽，每柱高 = `value/max × 100%`（非零至少 8% 高让
    单柱可见；零柱占位 2% 灰色 dashed-like 视觉）
  - sky 蓝色（与 PanelStatsCard "今日"数字同色）；零桶 muted border 色
  - 每柱 hover tooltip 显 "HH:00 · N 次"
  - 右侧 `Σ N` 显当日总和
  - 容器 hover 显总 tooltip "今日总 X 次主动开口"

## 验证

- `cargo check` clean
- `npx tsc --noEmit` clean
- 行为：
  - 入 PanelDebug → 24h bar 显（首次抓取约 50-100ms 后）
  - 当日开口集中在午餐 / 下午时段 → 12 / 14 / 15 等柱明显高
  - 零桶（如 03:00-05:00）显灰色矮柱 + 0.4 opacity 占位
  - 每柱 hover tooltip 显时间 + 次数
  - 60s 后自动 refresh（用户新触发了一次 proactive 后 1 分钟内能看到新柱
    长出来）
  - 当日完全没主动开口 → 全部柱矮灰 + Σ 0

## 不在本轮范围

- 没做"自定义日期回看"：当前只显 today；想看"昨天的节奏"可作 day picker
  后续需求
- 没做 stacked bar（按 motion 分色）：本轮 1 维计数已经够直观；多维要扩
  后端聚合
- 没把 24h bar 推到 PanelStatsCard：那卡是固定 stat strip 与时间序列概念
  分离；新加一行更清晰
