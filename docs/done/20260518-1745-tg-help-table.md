# TG `/help_table` 命令（iter #582）— audit family 分组速查表

## Background

近 30 iters 加了 30+ audit 系列命令（pin / cat / rename / idle / streak
/ speech × today/yesterday/thisweek / 7d/30d 等矩阵）。命令总数 ≥ 100，
新用户 / 老用户 jog memory 时翻 flat `/help` 一行描述全表低效。

本 iter 加 `/help_table` 按主题分组列举，让 owner 快定位「这个 audit
在哪个家族」后再 `/help <cmd>` 看细节。

## Changes

6-surface 同步：

1. Enum `HelpTable`（紧贴 `RecentRenames`，与 `Help` 同 system 类）
2. name / parser / no-args title chain / registry en+zh / help-detail /
   help-table / 三份 drift lists
3. 新 pure `format_help_table_reply()` 硬编码 family 分组（无参数无 IO）
4. Handler 直接调 formatter

## Key design decisions

- **硬编码 family 分组**：与 /help (registry-derived flat) 不同 — 本表
  是策展视角。新加命令时需手动更新 family map — tradeoff for curation
  quality
- **emoji 与既有 chip family 配色 / 含义协调**：pin = 📌 / cat = 🌱
  / rename = 🔁 / idle = 💤 / stale-warning = ⚠️ — 视觉 family 一致
  跨 surface（PanelTasks chip-bar / PanelMemory chip / TG）
- **顺序：高频常用在前**：pin / cat / rename / idle / streak 在前
  （近月主力 audit 系列），speech / alarm / status 在中，batch /
  system 在后（少用 / dangerous）
- **末尾教学指 /help 兄弟入口**：避免 owner 困惑「这是替代 /help 还是
  补充」— 明示三入口分工：flat / 单命令详细 / 全文搜
- **2 unit tests**：parser（含尾部多余 args 容忍）+ formatter spot
  check（必含所有 13 family group label + 12 代表性命令 + 相关教学
  链接）

## Future iters (out of scope)

- **registry-derived auto-grouping**：每条命令加 `family: AuditFamily`
  field，formatter 按 family 自动 group — 解决「新命令手动更新 map」
  痛点。需 ALL_HELP_TOPICS 表 schema 扩展；中型 refactor
- **`/help_table <family>`**：只显单 family 详细列表 + 一行描述。比
  /help <cmd> 更聚焦 — 「pin 系列全谱用法」单命令出 audit 答案
- **i18n**：当前中英混杂；统一中文为主 — 与既有 help-detail 一致
