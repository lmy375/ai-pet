### `reconstruct_alias_chain(history_lines, current_title) -> Vec<(ts, old)>`

pure helper — 输入完整 butler_history 行列表 + 当前 title：

```rust
let mut chain: Vec<(String, String)> = Vec::new();
let mut head_title = current_title.to_string();
// reverse iterate events; for each action=="rename" + title==head_title
// extract [was: old] → push (ts, old); head_title := old; continue
for (ts, action, title, snippet) in events.iter().rev() {
    if action.to_ascii_lowercase() != "rename" { continue; }
    if *title != head_title { continue; }
    let Some(old) = extract_was_from_snippet(snippet) else { continue };
    if old == head_title { continue; } // defensive
    chain.push((ts.clone(), old.clone()));
    head_title = old;
}
chain.reverse(); // 旧→新 alias 顺序
```

## Key design decisions

- **单向 walk（从 current 倒查 [was: X]）**：一次扫即可。比双向 bfs
  / dfs 简单；renames 严格 new←old 链，无需正向扫
- **`if old == head_title` defensive**：rename to self 不应发生但兜底
  防无限循环
- **chain.reverse() 后输出**：用户最熟「最早 → 最新 → 现在」时序
  阅读
- **format_timeline_ts 复用**：与 /timeline 行 ts 格式（MM-DD HH:MM）
  一致；视觉一致让 /timeline + /aliases 并用时 ts 对得上
- **教学兜底指 /timeline 而非 /show**：/timeline 含 rename 行（iter
  #568 已 wire）— 与 /aliases 互补，更对应「想看 task 历史」需求
- **6 unit tests**：parser + 4 reconstruct cases（empty / 单 rename /
  多 rename / 不相关 rename 不污染）+ 2 formatter（empty fallback +
  multi-row 带 → 现叫 X 尾）
