### Generic formatter: `format_cat_decay_reply(rows, threshold_days)`

旧 `format_cat_decay_7d_reply(rows)` 重构为 `format_cat_decay_reply(rows,

旧 test 路径。

## Key design decisions

- **shared compute helper**：避免「30d handler 是 7d handler 的 copy
  paste」— 阈值参数化让未来 `cat_decay_Nd` 加变体不再增重复代码。同
  时让 sort tie-break 等细节单点维护
- **formatter 参数化**：与 compute 同样精神 — header 文案 / 兜底教学
  「N 天」字段统一参数。30d 兜底里仍指 /cat_growth_7d（不分 7d/30d 因
  growth 仅 7d 一档；可未来加 cat_growth_30d）
- **旧 API 保留为 wrapper（`#[allow(dead_code)]`）**：本 iter 的重构
  对老测试 path 透明。/cat_decay_7d 仍可独立调 wrapper，调用路径未来
  随重构清理
- **/cat_growth_30d 不顺手实施**：reverse 对偶可能 propose 但 growth
  数据 substrate 比 decay 简单（count vs max ts），单独做容易；不强
  挤入本 iter
- **3 unit tests**：parser + 30d threshold 注入 header + 30d 阈值
  注入兜底教学。共享 helper 的 sort / filter 逻辑已经被既有 cat_decay_7d
  multi-row test 覆盖（同 algorithm 不需重测）
