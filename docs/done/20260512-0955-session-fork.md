# PanelChat session ⑂ Fork 按钮

## 需求

聊天到一半想"基于这段对话分叉新讨论"（如把某条 assistant 回复当做基础
再展开 / 想试不同方向但不污染原 session）只能：1）手动新建空 session 2）
来回切粘内容。加 ⑂ Fork 按钮一击 fork 当前 session 末尾 N 条到新 session。

## 实现

`src/components/panel/PanelChat.tsx`：

- 新 `handleForkSession(keepN)` useCallback：
  - invoke `create_session` 拿新 session id
  - source items = `items.slice(items.length - keepN)`（keepN=Infinity 全
    复制）
  - source messages = system message 保留 + 末尾 keepN 条非 system
    （与压缩历史同算法）
  - 新 session 的 title = `${sessionTitle} · fork`（限 60 字符）
  - `save_session` 写新 session；`list_sessions` 刷 list；`loadSession`
    切到新 session
  - 反馈走 `exportToast`（"已 fork N 条到新 session 「...」" / "fork 失败：..."）
- 新 `forkPopoverOpen` state + outside-click / Esc 关 effect（setTimeout(0)
  挂 mousedown 避免触发即关）
- session bar：⑂ Fork 按钮（仅 items.length > 0 时显）+ popover：
  - 三档：整段复制 / 近 20 / 近 10
  - 副标显具体数量（如"末段 20"，items.length < 选项数时显"（不够，全部）"）
  - 底部提示"fork 后自动切到新 session；原 session 不动"

## 验证

- `npx tsc --noEmit` clean
- 行为：
  - 空 session → ⑂ Fork 按钮不显（与"+ 新会话"语义重复时退场）
  - 30 条 session 内点 ⑂ → popover 显 "整段 30 条 / 近 20 末段 20 / 近 10 末段 10"
  - 点"近 10 条" → 创建新 session 含末尾 10 items；自动切过去
  - 新 session 名 `<原名> · fork`，tab 栏 / dropdown 即刻显
  - 原 session 不变（items / messages 都还在）
  - LLM 在新 session 看到的 prompt = system + 末尾 10 条 messages，可继续
    对话延续语境
  - Esc / 外部点击 popover 关
  - fork 失败（罕见 IO 异常）→ exportToast 显错误

## 不在本轮范围

- 没做"先选定从哪条开始 fork"（cursor 选起点）：popover 已够覆盖常见的
  "近 N"诉求；任意切点需要每条 bubble 加按钮，UI 杂
- 没把 fork 跟 export snapshot 联动：fork 本身就是数据落盘，可逆性已经
  靠"原 session 不动"覆盖
- 没在 mini chat 桌面气泡加 fork 入口：分叉是个 panel-side 重操作，桌面
  气泡是 quick-read，不必双入口
