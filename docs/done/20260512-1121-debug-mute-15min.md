# PanelDebug ⚙️ mute pet 15min 快捷按钮

## 背景

调 prompt / 改 SOUL.md / 测临时 prompt 时，主动开口节奏会被 proactive 触发打断（提示音 + 弹窗），需要的是"接下来 15 分钟别吵"。

现状路径：PanelChat 输入 `/sleep 15`。在 PanelDebug 里跑测试时切回 chat 再返回 debug 的成本不低。

## 改动

PanelDebug 顶部 proactive 工具栏（"立即开口 / ✏️ 临时 prompt / 看上次 prompt / DevTools / 🧹 重置 stash"）末尾加一个按钮：

- **未 mute 时**显示 `⚙️ mute 15min`，点击后 invoke `set_mute_minutes(15)`。
- **已 mute 时**显示剩余分钟数 `🔕 mute 14m` 并允许再点解除（`set_mute_minutes(0)`）。
- 复用现有 `set_mute_minutes` / `get_mute_until` 后端命令（PanelChat /sleep 同源）。

## 状态/轮询

- 挂载时 + 每 30s polling `get_mute_until` —— 与按钮显示文字同步。
- 点击后立即本地刷新状态，不等下一轮 poll。
- 兜底：解除失败时复用 `proactiveStatus` 反馈条显错。

## 不做

- 不做 5min / 30min / 1h 多档（先就一个常用值）—— 用户要更长直接用 /sleep。
- 不做"自动 unmute 后 toast"—— 让按钮本身的 polling 文字变化即可。
- 不写测试 —— 纯 UI 按钮，依赖后端已有命令，没有需要 pin 的"真实行为"。

## 验收

- 点击 ⚙️ mute 15min → 按钮文字变 🔕 mute 15m
- 等 1 分钟 → 文字自动刷成 🔕 mute 14m
- 点击 🔕 mute → 文字回到 ⚙️ mute 15min（mute 解除）
- 期间 proactive 不再 fire（与 PanelChat /sleep 同语义验证）
