# PanelTasks 任务卡详情区 raw_description > 300 字折叠

## 需求

raw_description 是 index.yaml 里原始字符串：含 `[task pri=...]`、
`[origin:...]`、多条 `[result: ...]`、`[done]`、`[error: ...]` 等 marker
+ 用户写的本体。LLM 长跑后这段常超过千字 —— 展开任务详情时一气推
屏，挤掉下面的进度笔记 / history 视图。300 字折叠 + 展开按钮让默认
视图保持轻量。

## 实现

`src/components/panel/PanelTasks.tsx`：

- 新 state `expandedRawDescTitles: Set<string>` + `toggleRawDescExpand`
  helper —— 同时多个任务可独立展开 / 折叠
- 不跨 session 持久化：每次重开 panel 默认回到折叠态，避免"全展开"
  视觉爆炸
- 详情区 raw_description box 改成 IIFE：
  - `LIMIT = 300`，`isLong = raw.length > LIMIT`
  - 默认显前 300 字 + `…`；展开后显全文
  - `isLong=true` 才浮"展开 / 收起"按钮，短描述与原视觉一致
  - 按钮文案三态：`… 展开剩余 N 字` / `收起 (N)` —— 数字明确避免用
    户折前疑惑"折掉多少"

视觉跟 PanelMemory 的"每日小结展开 / 收起"按钮 (R143)、PanelTasks
的"history 折叠"按钮同款 —— accent 色 + 无 border + 单行行内 button。

## 验证

- `npx tsc --noEmit`：clean
- 行为：
  - raw_description 80 字 → 全文显，无按钮
  - raw_description 500 字 → 折到 "前 300 字…" + "… 展开剩余 200 字" 按钮
  - 点击展开 → 显全文 + "收起 (500)"
  - 再点击 → 折回
  - 同时展开任务 A 和 B（B 不影响 A 的状态）
  - 关 panel 重开 → 全部回到折叠态（无持久化）

## 不在本轮范围

- 没把阈值（300）做成可配：300 是经验值；大多数 raw_description 在
  100-200 字，> 300 即"长 marker 堆积"。可配化成本不值
- 没做 markdown 渲染（粗体 / 列表）：raw_description 是原始 yaml 串，
  不该解析；展示 monospace + pre-wrap 让 markers 可读保留
- 没在折叠态把 marker 剥掉（只显本体）：那是 chip 的职责（done / error
  chip 已经把对应 marker 视觉化）；折叠态只是裁字符数，不做语义剥离
- 没让折叠态自动 highlight markers（如 [error: ...] 标红）：本轮 scope
  小，markers 在阅读态用 mono 字体已能视觉区分
