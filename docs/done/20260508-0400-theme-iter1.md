# 深色 / 浅色主题（迭代 1）— 设计令牌系统 + 切换 hook

> 对应需求（来自 docs/TODO.md）：
> 抽 design token 模块 `src/theme.ts`，定义 light / dark 两套；PanelApp 顶部加 ThemeProvider + localStorage 偏好。本迭代仅引入 token 系统 + 主题切换 hook，不动既有 inline 颜色 — 后续迭代逐面板替换。

> GOAL.md 已把 "做深色 / 浅色主题" 加入约束 → 本工作分多迭代落地，避免单
> 迭代触动太多 inline color 引入回归。

## 目标

PanelTasks / PanelDebug 等面板有大量 inline `style={{ background: "#fff", ... }}`
直接写死十六进制色，要做夜间主题必须先有"统一令牌入口"。本轮：
1. 新建 `src/theme.ts` 定义 light / dark 两套核心 token（bg / fg / muted /
   border / card / accent 等基础 6-8 个）
2. CSS 变量方案（不走 React Context）：`applyTheme(t)` 在
   `document.documentElement` 上 `setProperty(--pet-color-bg, ...)` 等
3. localStorage 持久化偏好（`pet-theme: "light" | "dark"`）
4. PanelApp 启动时 read + apply
5. 顶部窗口顶 toolbar 加一个 toggle 按钮（☀ / 🌙），点击切换 + 持久化
6. **示范用法**：把 PanelApp 顶层 `<div>` 的 background / 顶部 tab bar
   的 background / borderBottom 改成 `var(--pet-color-bg)` 等 — 让用户
   切换时看到顶部框架在 light/dark 间切换；inner panel 内容暂保留原色
   （后续迭代逐 panel 替换）

## 非目标

- 不替换 PanelTasks / PanelDebug / PanelPersona / PanelChat 内的 inline
  color。那是 50+ 处的大改动，留迭代 2+ 按 panel 切。
- 不做"系统跟随" `prefers-color-scheme`。先支持手动 light / dark 切换；
  系统跟随是迭代 3+ 加 `auto` 三态。
- 不做主题切换的过渡动画。CSS var 变化即时生效；动画是装饰性需求。

## 设计

### token 集合

最小集合（6 个），覆盖框架级 surface：

```ts
export type ThemeTokens = {
  bg: string;       // 页面 / panel 容器底色
  card: string;     // 卡片 / formCard 内层底色
  fg: string;       // 主要文本
  muted: string;    // 次要文本 (hint / placeholder / count)
  border: string;   // 边框 / 分隔线
  accent: string;   // 主品牌色（active tab、primary button），跨主题保持一致
};
```

### CSS 变量名

`--pet-color-bg` / `--pet-color-card` / `--pet-color-fg` / `--pet-color-muted`
/ `--pet-color-border` / `--pet-color-accent`。前缀 `pet-` 防止与全局 / 第三
方变量冲突。

### light vs dark 取色

light（与现状对齐）：
- bg: #f8fafc
- card: #fff
- fg: #1e293b
- muted: #64748b
- border: #e2e8f0
- accent: #0ea5e9

dark：
- bg: #0f172a
- card: #1e293b
- fg: #f1f5f9
- muted: #94a3b8
- border: #334155
- accent: #38bdf8（dark 下提亮一档保对比度）

### theme.ts API

```ts
export type Theme = "light" | "dark";
export const TOKENS: Record<Theme, ThemeTokens>;
export function applyTheme(t: Theme): void;  // 写 CSS 变量
export function getStoredTheme(): Theme;     // localStorage / default
export function setStoredTheme(t: Theme): void;
```

无 React Context — 用 `useTheme` hook 内部用 `useState` + module-level
event emitter（极简，避免 Context 模板代码）。或最简：让组件用 `useState`
+ 显式调 setStoredTheme + applyTheme + setState 三件套；不抽 hook 也可。

最简：暴露一个 `useThemeToggle` hook 自封装这三步。

### PanelApp 接入

启动时 `applyTheme(getStoredTheme())`；顶部 tab bar 旁加 ☀ / 🌙 toggle
按钮：点击 `setStoredTheme(next)` → `applyTheme(next)` → setState 触发 re-render。

### 框架级颜色应用

把 PanelApp 顶层 `<div>` 的 inline `background: "#f8fafc"` 改为
`background: "var(--pet-color-bg)"`；tab bar 的 `background: "#fff"` →
`var(--pet-color-card)`，`borderBottom: "1px solid #e2e8f0"` →
`1px solid var(--pet-color-border)`，inactive tab 字色 `#64748b` →
`var(--pet-color-muted)`，active tab 字色 `#0ea5e9` →
`var(--pet-color-accent)`。

### 测试

`theme.ts` 是模块入口，applyTheme 操作 DOM —— 单测意义不大（要 jsdom）。
不写单元测试；通过 tsc + 手测覆盖：切换按钮 → 顶部 bg 变深 → 重启 panel
后保持深色。
