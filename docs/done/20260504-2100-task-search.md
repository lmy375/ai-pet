# 任务搜索 — 开发计划

> 对应需求（来自 docs/TODO.md「已确认」）：
> 任务搜索：面板「任务」标签加搜索框，按 title / body 关键字过滤，与标签筛选互补 —— 不带 # 的关键词也能定位。

## 目标

在 `PanelTasks.tsx` 顶部加一个搜索框，按用户输入的关键字（不区分大小写）即时过滤可见任务。匹配范围：`title` 与 `body`（剥过 origin/result 的展示态）。空查询时不过滤。

与已有「显示已结束」开关复合：搜索 ∩ 已结束筛选都通过的任务才展示。

## 非目标

- 不接 backend 全文索引 —— 任务列表典型 < 30 条，前端 in-memory filter 足够。
- 不做高亮命中片段 —— v1 只过滤 / 显示，命中位置可视化等用户反馈再加。
- 不扩展到搜索 detail.md 内容 —— 后端 `memory_search` Tauri 命令已能跨记忆类目搜索，需要时用户走那条路。
- 不做 tag 联合 —— `#xxx` 关键字理论上能命中 description 里的 tag，但本面板 tag 已用专门的 chip 展示；搜索就是简单的子串匹配。

## 设计

### 状态

`PanelTasks` 加 `const [search, setSearch] = useState("")`。

### 过滤逻辑

```ts
const visibleTasks = tasks
  .filter((t) => showFinished || !isFinished(t.status))
  .filter((t) => {
    const q = search.trim().toLowerCase();
    if (!q) return true;
    return t.title.toLowerCase().includes(q) || t.body.toLowerCase().includes(q);
  });
```

顺序：先 status 过滤再关键字过滤 —— 任意一边都可在零成本下早退。

### UI

搜索框在「队列（按宠物处理顺序）」section title 下面、`<label>显示已结束</label>` 切换之上。普通 `<input type="text">`，placeholder 为「按标题或内容搜索…」。右侧加一个「✕」清除按钮，仅 `search.length > 0` 时显示。

输入框 width: 100%，与表单其他 input 风格统一（borderRadius 4, border 1px #e2e8f0）。

### 空态文案分裂

`visibleTasks.length === 0` 时空态文案要分清楚原因：
- 有 search 且无命中 → 「没有匹配 "{search}" 的任务」
- search 空 + showFinished → 「还没有任何任务」
- search 空 + !showFinished → 「队列里没有进行中的任务」（已有逻辑）

### 数据流

无后端变化。`task_list` 仍返回全量；前端在 setState 时不动，仅在渲染层 derive。
