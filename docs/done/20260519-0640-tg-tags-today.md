# TG bot 加 `/tags_today` 命令（iter #542）

## Background

`/tags` 列**全量**用过的 #tag 计数（top 15）。但常用场景是「今天我在做
什么主题」 — 全量 lifetime tag 矩阵太广，看不出本日聚焦点。

本 iter 加 `/tags_today` — /tags 的今日切片。让 owner 看「今日动过 task
含哪些 tag」。

与 `format_tags_reply` 差异：

- 仅扫今日 updated_at（prefix 10 字符日期比较）
- header 显日期 scope 而非「共 N 个 tag」
- **无 cap**（今日范围天然小 — 全量 /tags 用 15 cap 是因为 lifetime tag
  可能很多；today 通常 < 10 个不同 tag，cap 反而漏看）
- 空集兜底教学指 /tags 全量入口（避免循环）

## Key design decisions

- **clone 不抽 generic**：与既有 today/yesterday/thisweek split 模板
  一致 — 单测点稳定 + 行内 < 50 行
- **无 cap**：今日 tag 数量小；cap 反而错过 audit 信号
- **prefix 10 字符日期比较**：与 /touched_today / /alarms_today / etc.
  同协议
- **untagged 计数保留**：与 /tags 同 — owner 想看分母（多少 today task
  没分类）有 audit 价值
- **`b.1.cmp(&a.1)` 不引 secondary tie-break**：BTreeMap 默认 key asc，
  ties 时字典序保结果稳定（与 /tags 同 sort）
- **5 个 unit tests pin 真实行为**：parser（含尾部 token 容忍）+ 空集
  兜底（验 /tags 教学不指自身）+ today + tag aggregate（验日期过滤
  正确，count 累加）+ untagged 计数（非 0 时显）+ desc by count 排序
