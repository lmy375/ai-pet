# ChatMini streaming 中"Esc 取消"hint

## 需求

上一轮加了 Esc soft-cancel，但快捷键不可见 —— 用户得读 dev log 才知道。在
streaming bubble 旁边浮个小 chip 提示，让快捷键自发现。

## 实现

`src/components/ChatMini.tsx`：streaming bubble 容器从 `flex justifyContent
flex-start` 加 `alignItems: flex-end + gap: 4`，让旁边能挂一个等宽小 chip。

chip 仅在 `onCancel` 注入时渲染（不注入时表示 caller 没接 cancel，hint 是
噪音）：

```tsx
<span
  style={{
    fontSize: 10,
    color: muted,
    background: card,
    border: 1px solid border,
    borderRadius: 4,
    padding: "1px 6px",
    pointerEvents: "none",
    fontFamily: mono,
  }}
  title="按 Esc 取消生成（已写出的内容保留）"
>
  Esc 取消
</span>
```

`pointerEvents: none` 让 chip 不挡用户操作；mono font 区分于正文气泡的中文
对话语气；放在 bubble 同行右侧 baseline 对齐，stream 滚动时自然跟随。

## 验证

- `npx tsc --noEmit` clean
- 行为：
  - 桌面发一个长 prompt → streaming bubble 出现后旁边浮"Esc 取消"chip
  - tooltip"按 Esc 取消生成（已写出的内容保留）"补充语义
  - stream 完成（done）→ showStreamingBubble false → chip 同步消失
  - 用户按 Esc → cancel 触发，chip 也消失（isLoading false）
  - onCancel 未传时 chip 不显（默认 props 安全）

## 不在本轮范围

- 没做"Esc 倒计时"等动态视觉 —— 一个静态 chip 已足够指向快捷键
- 没改 ChatPanel textarea：那里 Esc 已映射到清空输入；不在 textarea 时 Esc
  才取消生成，hint 准确反映这点
