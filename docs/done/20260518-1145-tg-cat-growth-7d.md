# TG bot 加 `/cat_growth_7d` 命令（iter #557）

## Background

PanelMemory 已有 per-cat header「📊 7d +N」chip（iter #555）— 单 cat
delta 视角。但 TG 端缺 cross-cat 排行视图：「我这周哪类知识投入最多」
audit。本命令把 7d 净增按 desc 排，让 owner 远程一眼看跨 cat 活跃度
分布。

## Changes

按 6+ surface 同步：

1. Enum `CatGrowth7d`（无参；紧贴 `RandomPinned`）
2. name / title-noargs / parser / registry en+zh / help-detail /
   help-table / 三份 drift lists

### `format_cat_growth_7d_reply(rows: &[(key, label, delta)]) -> String`

pure formatter — caller 已 sort desc + filter delta > 0：

- 空 rows → 教学指 /touched_thisweek（task 维度本周复盘） +
  /find <kw>（不限日期）；不指 self / 不指 PanelMemory chip（cross-surface
  教学不合适）
- 非空 → header `🌱 近 7 天各类新增（共 N 条 across M cats）：` + 每
  行 `· <key> (<label>) · +<delta>`
- **label == key 或空 → 仅显 key**（避免 `butler_tasks (butler_tasks)`
  视觉冗余）

### Handler (bot.rs)

紧贴 RandomPinned：

```rust
TgCommand::CatGrowth7d => {
    let now_ms = chrono::Local::now().timestamp_millis();
    let cutoff_ms = now_ms - 7 * 24 * 60 * 60 * 1000_i64;
    let mut rows: Vec<(String, String, usize)> = Vec::new();
    if let Ok(index) = crate::commands::memory::memory_list(None) {
        for (key, cat) in index.categories.iter() {
            let mut delta = 0usize;
            for it in &cat.items {
                if it.created_at.is_empty() { continue; }
                let Ok(t) = chrono::DateTime::parse_from_rfc3339(&it.created_at)
                else { continue; };
                if t.timestamp_millis() >= cutoff_ms { delta += 1; }
            }
            if delta > 0 { rows.push((key.clone(), cat.label.clone(), delta)); }
        }
    }
    rows.sort_by(|a, b| b.2.cmp(&a.2).then_with(|| a.0.cmp(&b.0)));
    format_cat_growth_7d_reply(&rows)
}
```

## Key design decisions

- **数据源 memory_list(None)**：read 路径已自动用 SQLite 真相覆盖
  `butler_tasks` / `todo` / `task_archive` 段（与 PanelMemory 同源）
  — 不会算到 yaml orphan
- **delta > 0 才列**：与 PanelMemory chip 同决策 — 0 净增 cat 不挂，
  避免 stale cat 输出噪音。owner 关心「谁在长」而非「谁没动」
- **delta desc + key asc tie-break**：稳定输出便于测 + 视觉上活跃 cat
  优先
- **7d 滚动窗口**：与 PanelMemory chip 一致 — 滚动比自然周稳；避免周一
  归零的"假死"信号
- **label == key 仅显 key**：避免 `butler_tasks (butler_tasks)` 冗余 —
  许多 cat label 与 key 同（系统 cat），少数才有人类可读 label
- **空集兜底教学**：指 /touched_thisweek + /find — 这是 sibling scope
  alt（task 维度本周 vs memory 维度 7d）。不指 PanelMemory chip 因为
  cross-surface 教学（TG → desktop）容易引发误解
- **4 unit tests**：parser（含尾部多余 args 容忍）+ empty fallback +
  multi-cat desc render（顺序 + total + 行格式断言）+ label==key 不
  冗余兜底
