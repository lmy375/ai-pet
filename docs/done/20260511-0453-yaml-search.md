# 设置页 raw YAML 模式加搜索

## 需求

config.yaml 长起来（mcp_servers / telegram / proactive / morning_briefing /
memory_consolidate / chat / motion_mapping…）后，raw 模式手动滚到目标字段费
劲。需要一个简易的"输入 → 跳到首匹配，Enter / ⌘G 反复跳"。

## 设计

不做 overlay 高亮（textarea 不支持 inline styling，套 absolute pre 既要同步
scroll 又要算字体度量）。用 textarea 原生 selection：找到匹配位置 →
`setSelectionRange + scrollTop`，selection 自身高亮即结果反馈，体验等价于浏览
器 Cmd+F 在普通 textarea 上的表现，零额外渲染层。

## 实现

`src/components/panel/PanelSettings.tsx`：

- `rawYamlRef = useRef<HTMLTextAreaElement>` 加在组件顶部，传给 raw 模式
  textarea 与新 `<YamlSearchBar>` 子组件
- `<YamlSearchBar>` 子组件（同文件，函数声明）：
  - props：`text` + `textareaRef`
  - state：`query`、`activeIdx`
  - matches: useMemo 算所有匹配位置（lower-case 不敏感、含重叠安全的 `i+max(1,len)` 步进）
  - useEffect on `query` 变化：reset activeIdx + 自动跳首匹配
  - `goNext` / `goPrev`：循环往返 + jumpTo
  - `jumpTo(idx)`：textarea.focus + setSelectionRange + 估算行号设 scrollTop（lineHeight ≈ 19.2px，上面留 3 行 padding）
  - 键盘：Enter / ⌘G → next；Shift+Enter / ⌘⇧G → prev；Esc → 清空 query
  - UI：input + N/M 计数 + ↑ ↓ 按钮

PanelSettings 的 raw 模式渲染在 textarea 上方挂 `<YamlSearchBar text={rawYaml} textareaRef={rawYamlRef} />`。

## 验证

- `npx tsc --noEmit` clean
- 行为：
  - raw 模式下输 `mcp_servers` → 自动跳到 first match，selection 高亮
  - Enter → 跳到 next match（循环）；Shift+Enter → prev
  - ⌘G / ⌘⇧G → 同
  - Esc → 清空 query，selection 保留在最后位置
  - 0 匹配 → "0 匹配" 文案 + 按钮 disable
  - query 没改但 text 变化 → 不重跳（避免编辑时光标被抢）

## 不在本轮范围

- 没做"全部高亮"模式：要 overlay <pre>，目前 native selection 已能让用户专注当前匹配
- 没做正则 / 大小写敏感切换：raw YAML 字段都是 ASCII，简单 substring 足够；如有用户反馈再加
- form 模式已有的 `searchQuery` 是另一套（按 section 标题 + 关键字过滤），不冲突
