# TG `/cat_decay_7d` 命令（iter #564）— /cat_growth_7d 反向 stale 审计

## Background

/cat_growth_7d（iter #557）audit「哪类知识在长」 — 正向活跃度。反向
缺：「哪类已停滞 — 该 archive 或重投入」。完整 audit 矩阵需要双向：

- /cat_growth_7d ← 正向：近 7d 有新增的 cat
- /cat_decay_7d ← 反向：近 7d 完全没 update 的 cat

## Detection

```
stale = ∀ items.updated_at, max(updated_at) < now - 7d
empty cat (0 items) is skipped — 没数据时「未动」无语义
```

按「最后 update 的 ts asc」(即 days_since desc) 排 — 最老 stale 在上，
owner 先看最该处理的。

## Changes

按 6+ surface 同步：

1. Enum `CatDecay7d`（紧贴 `CatGrowth7d`）
2. name / parser / no-args title chain / registry en+zh / help-detail /
   help-table / 三份 drift lists

### `format_cat_decay_7d_reply(rows: &[(key, label, days)]) -> String`

pure — caller 已 filter + sort：

- 空 → 「健康状态 — 所有 cat 7d 内都有动」+ 教学指 /cat_growth_7d 正向
- 非空 → header「🍂 近 7 天 0 update 活动的 cat（共 N 条 stale）」+
  每行 `· <key> (<label>) · 最后 update <N> 天前`
- label == key 或空 → 仅显 key 避免冗余（同 /cat_growth_7d formatter
  pattern）

### Handler (bot.rs)

紧贴 CatGrowth7d：

```rust
TgCommand::CatDecay7d => {
    let now_ms = chrono::Local::now().timestamp_millis();
    let cutoff_ms = now_ms - 7 * 24 * 60 * 60 * 1000_i64;
    let mut rows: Vec<(String, String, i64)> = Vec::new();
    if let Ok(index) = crate::commands::memory::memory_list(None) {
        for (key, cat) in index.categories.iter() {
            if cat.items.is_empty() { continue; }
            let mut max_u: Option<i64> = None;
            for it in &cat.items {
                // parse updated_at → ms；累积 max
            }
            if let Some(max_ms) = max_u {
                if max_ms < cutoff_ms {
                    let days = (now_ms - max_ms) / 86_400_000;
                    rows.push((key.clone(), cat.label.clone(), days));
                }
            }
        }
    }
    rows.sort_by(|a, b| b.2.cmp(&a.2).then_with(|| a.0.cmp(&b.0)));
    format_cat_decay_7d_reply(&rows)
}
```

## Key design decisions

- **empty cat 跳过**：0 items 时"未 update"无语义 — owner 没在那个
  cat 里放任何东西，stale 不应被列。consolidate / 新建后未填的 cat
  不会进 false positive
- **全 items 解析失败也跳过**：脏数据兜底 — `updated_at` 字段非
  RFC3339 时该 cat 不被误判为 stale
- **days desc + key asc tie-break**：稳定输出 + 最老 stale 在上 — 让
  owner 优先看「最该处理的」
- **「健康」兜底而非「空」**：empty result 是好消息（所有 cat 都活
  跃）— 文案 frame 为「健康状态」而非「空」更准信号传递
- **不指 /touched_thisweek / /pinned_drop_7d 兜底**：不同 axis 的
  audit — task 维度 vs cat 维度。只指 /cat_growth_7d 正向 sibling
- **3 unit tests**：parser + healthy empty fallback + multi-row desc
  + label==key 不冗余（覆盖 formatter 关键分支）

## Recurring lesson — ASCII quotes break Rust strings

Help-detail 内一开始用 ASCII `"..."` 包专有名词，导致 Rust 字符串
literal 被破坏（曾在多个 iter 重复犯，这次又踩）：

- `"创建后从未维护过"` → 修为 `「创建后从未维护过」`
- `没数据时"未动"无意义` → 修为 `没数据时「未动」无意义`

下次写 help-detail 长字符串时**首先检查所有 ASCII 引号**，全角化。
