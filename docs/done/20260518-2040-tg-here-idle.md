# TG `/here_idle` 命令（iter #592）— stale → transient_note pipeline

## Background

iter #591 加了 /here_pin（pinned → transient）— 让 pet reply 聚焦
pinned。本 iter 补 stale 维度对偶 /here_idle — 让 pet 主动提醒 owner
处理 stale backlog。

完成 here-* 注入命令双视角：
- /here_pin：正向关注 — 「这是我此刻在乎的，多帮我推进」
- /here_idle：反向警示 — 「这是我搁着没动的，帮我决策」

## Changes

6-surface 同步（与 /here_pin 镜像）：

1. Enum `HereIdle`（紧贴 `HerePin`）
2. name / parser / no-args title chain / registry en+zh / help-detail /
   help-table / 三份 drift lists
3. 新 pure `format_here_idle_reply(rows: &[(title, days)], until_local)`
4. Handler scan chat-scoped views filter pending + updated_at ≤ now-7d
   + sort idle days desc → 拼紧凑 stale context 字串 → set_transient
   _note 60min
5. /help_table idle 家族 + family detail map 同步加入 /here_idle

## Output

```
💤 已注入 N 条 idle task 到 transient_note（到 HH:MM 失效）
· 「写周报」（idle 14 天）
· 「整理 Downloads」（idle 9 天）
...
```

空 → 「无 7d+ idle pending — 健康状态」+ 教学指 /idle_7d。

## Key design decisions

- **镜像 /here_pin 实现**：复用 set_transient_note + 紧凑 inline-text
  注入 pattern；差仅 filter 条件（pinned vs idle 7d+）
- **idle days 显在 title 后**：让 pet 看到 transient_note 时知「这条
  搁了多久」— prompt 信息含量大于纯 title list
- **idle 天数 desc 排序**：最老 stale 在上 — 与 /idle_7d 一致
- **空时「健康状态」frame**：与 /cat_decay_7d 兜底风格一致 — empty
  是好消息不该 dead chip
- **transient_note text 前缀 「💤 stale context（>7d idle）」**：让
  pet 系统 prompt 注入时一眼识别这是 idle context（vs /here_pin 的
  📌 pin context）— 两路径 prompt 上下文可区分
- **3 unit tests**：parser + 空兜底 + multi-row（含 idle days label）
