# PanelChat textarea Esc 清空（Iter R127）

> 对应需求（来自 docs/TODO.md）：
> PanelChat textarea Esc 清空：非 slash 模式下 textarea 非空 + Esc → 清空 input（slash 模式 Esc 已被既有 logic 用 — 关命令面板）。与 R110 PanelMemory modal Esc / R96 搜索 panel Esc 习惯一致。

## 目标

PanelChat 输入框已是 R126 改的 textarea。现在 Esc 仅在 slash 模式有效
（line 669-：清掉 `/` 命令前缀 + 关菜单）。非 slash 模式下打了一段长
prompt 改主意，没有快捷"取消重写"路径。

加 Esc 清空：非 slash 模式 + textarea 非空 + Esc → 清掉 input。空 input
按 Esc 不干扰（避免吞掉用户可能习惯的 IME / 浏览器 Esc）。

## 非目标

- 不持久化"已清掉"内容做撤销 —— Esc 清空是"我不发了"语义；要重新输
  用 ↑ 召回 lastSentMessage（已有 R126 同款 shell 风格）
- 不动 slash 模式 Esc —— 那已是"退出命令面板"语义，与本轮的"清空"不冲突
- 不动 IME composition / 输入法状态 —— Esc 在 IME 候选窗里是关候选词，
  那是浏览器/系统层处理；React onKeyDown 收到 Esc 时通常 IME 已关
- 不限定光标位置 —— 任意位置 Esc 都清整段（textarea 没"select all"必要）

## 设计

### handleInputKeyDown 非 slash 分支加 Esc

现有 ArrowUp 召回逻辑之前插：

```diff
 if (!slashMenuVisible) {
   if (e.key === "Enter" && !e.shiftKey) {
     e.preventDefault();
     submitInput();
     return;
   }
+  if (e.key === "Escape" && input.length > 0) {
+    e.preventDefault();
+    setInput("");
+    return;
+  }
   if (e.key === "ArrowUp" && input.length === 0 && lastSentMessage) {
     ...
   }
   return;
 }
```

`input.length > 0` 守卫让空 input 时 Esc 走默认（不拦截）；非空时清空 +
preventDefault 防止其它 listener 处理。

### 测试

无单测；手测：
- 输 "嗨" + Esc → input 清空
- 输 "/clear" + Esc → 走 slash 模式 Esc（现状不变）
- 空 input + Esc → 不动
- 多行输入 + Esc → 全清
- Esc 后再输 + Enter → 正常发送（无残留）
- Esc 后 ↑ → 召回 lastSentMessage（既有 ArrowUp 路径，需要 input 空才命中；
  Esc 后 input.length === 0，命中）
