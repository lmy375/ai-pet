# TG bot 加 `/mute_today` 命令（iter #519）

## Background

既有 mute 入口：
- `/mute N` — 静音 N 分钟（相对分钟数）
- `/sleep` — 一键 8h（晚安预设）
- `/sleep_until <HH:MM>` — 静音到目标时刻（可跨天）

但缺**「今夜不打扰」常用预设** — owner 在 23:00 / 22:30 想说「别再打扰
我，直到午夜」时心算「还多少分钟到午夜」是 friction。本 iter 加
`/mute_today` 直接处理。

#### 纯 formatter `format_mute_today_reply`

```rust
pub fn format_mute_today_reply(minutes: i64) -> String {
    let hours = minutes as f64 / 60.0;
    if minutes >= 60 {
        format!(
            "🌙 已静音 proactive 到本地午夜（00:00）— 还 {} 分钟（{:.1} 小时）",
            minutes, hours,
        )
    } else {
        format!(
            "🌙 已静音 proactive 到本地午夜（00:00）— 还 {} 分钟",
            minutes,
        )
    }
}
```

- ≥ 60 min：显「N 分钟（M.M 小时）」双格式 — owner 一眼看 hours
- < 60 min：仅 N 分钟 — 整数小时无意义（如「45 分钟（0.8 小时）」反而
  累赘）

### `src-tauri/src/telegram/bot.rs`

Handler 紧贴 CascadeRename 之前：

```rust
TgCommand::MuteToday => {
    use chrono::{Datelike, Local, TimeZone};
    let now = Local::now();
    let tomorrow = now.date_naive().succ_opt().unwrap_or(now.date_naive());
    let target = Local
        .with_ymd_and_hms(tomorrow.year(), tomorrow.month(), tomorrow.day(), 0, 0, 0)
        .single()
        .unwrap_or_else(|| now + chrono::Duration::hours(1));
    let minutes = (target - now).num_minutes().clamp(1, 1440);
    let _ = crate::proactive::set_mute_minutes(minutes);
    format_mute_today_reply(minutes)
}
```

边界防御：
- `succ_opt()` 跨月跨年 chrono 自动处理
- `clamp(1, 1440)` 防 DST 边界 / 时区抖动产生异常 minutes
- `.unwrap_or_else(|| now + 1h)` 防极端 DST gap（如 spring forward 时
  next midnight 可能落 invalid hour）— 至少给 1h fallback 让命令不无响应

## Key design decisions

- **复用 `set_mute_minutes` 同 /mute 路径**：与桌面 PanelDebug mute / TG
  /mute / /sleep / /sleep_until 同后端 — record_mute_engaged 让「🔕 今
  日 mute」chip 计数自动同步
- **clamp 1..=1440**：1 分（午夜前 1 秒触发兜底）/ 1440 = 24h（理论上
  本命令永远 < 24h，但 clamp 防 DST + timezone drift 边界）
- **decimal hours 仅 ≥ 60min 显**：低分钟数显「0.5 小时」反而累赘；
  整 hour cutoff 让 reply 行长稳定
- **succ_opt + with_ymd_and_hms**：标准 chrono 跨日 pattern，与既有
  /sleep_until 同 helper（line 1413+）— 一致 DST 处理
- **4 个 unit tests pin 真实行为**：parser（含 trailing tolerance）+
  formatter 3 路径（≥60 含 hours / < 60 仅 minutes / 边界 1 min）
- **不写 integration test**：handler 内 chrono::Local::now() 是 wall
  clock — 测试会因运行时刻不同失败。formatter 已 isolated 测过；
  handler 路径只是几行 chrono helper + 透传
