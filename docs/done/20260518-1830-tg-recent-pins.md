# TG `/recent_pins [N]` 命令（iter #585）— N-cap pin 决策 audit

## Background

完成 pin audit 矩阵：

| view | 7d window | N-cap |
|------|-----------|-------|
| forward (new pins) | /pin_grow_7d | **/recent_pins** (本 iter) |
| reverse (unpins) | /pinned_drop_7d | — |
| streak | /streak_pin | — |
| current snapshot | /pinned / /peek_pinned / /random_pinned | — |

owner 想看「最近 N 条 pin 决策不限日期」时缺工具 — /pin_grow_7d 是
7d 窗口，超出该窗口的 pin event 看不到。本命令按 N 数 cap 让 owner
浏览历史 pin 节奏。

## Algorithm

scan butler_history.log → 取 含 `[pinned]` snippet 行 → dedupe by
title 保留最早 sighting (= owner 首次钉它) → 按 ts desc 排 → cap N。

dedupe rationale：同 title 多次 update（pin 状态不变）只算 1 次「决
策事件」。这让 N=5 给出真正不同的 5 次决策而非 5 行可能都来自同 task。

```rust
let mut earliest_per_title: HashMap<String, String> = HashMap::new();
for line in content.lines() {
    if snippet.contains("[pinned]") {
        earliest_per_title.entry(title)
            .or_insert_with(|| ts_str.to_string());
    }
}
// content 是 chronological (oldest first)；or_insert 让首次保留 = earliest
let mut with_ts: Vec<(String, String)> = earliest_per_title.into_iter()
    .map(|(title, ts)| (ts, title)).collect();
with_ts.sort_by(|a, b| b.0.cmp(&a.0)); // RFC3339 lex desc = chron desc
with_ts.truncate(n as usize);
```

## Key design decisions

- **dedupe by title (earliest)**：避免同 task 多次 update 占用 N 个 slot；
  N=5 给出真正不同 task 的 5 次决策。tradeoff：unpin → re-pin 同 title
  时显示 1 行（earliest pin 时刻），后续 re-pin 不再 sighting — 数据
  substrate 不支持精确区分
- **不 filter 当前状态**：含已 unpin / done / archived 的 task — 「pin
  决策」是历史事件不必当前 active。与 /pinned_drop_7d 反向：那是「当前
  unpinned + 曾 pinned」；本命令是「曾 pinned, 不限当前状态」
- **RFC3339 字典序 == chronological**：`ts.cmp(&other_ts)` 安全 — ISO
  8601 ASCII prefix 单调，省 chrono parse
- **`format_timeline_ts` 复用 MM-DD HH:MM**：与 /recent_renames /
  /timeline / /aliases 一致让 ts 视觉一致
- **5 unit tests**：parser default + parser clamp（upper / lower）+
  empty fallback + multi-row + header total > shown
