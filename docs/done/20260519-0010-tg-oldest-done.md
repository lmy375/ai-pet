# TG bot 加 `/oldest_done [N]` 命令（iter #514）

## Background

`/recent [N]` 显最近 N 条 done（updated_at desc）— 「我最近完成了啥」。
反向视角缺：「最早完成的 N 条」— 「这条 task 我做了多久」考古 / 「最
老 backlog 终于完成」audit 场景。

四象限：

|        | 最近 (desc)   | 最早 (asc)    |
|--------|---------------|---------------|
| done   | /recent       | **/oldest_done** ← new |
| pending| /active_recent| /oldest_n     |

本 iter 补 done × asc 缺口，闭合 N-cap 视图矩阵。

## Changes

### `src-tauri/src/telegram/commands.rs`

按 6+ surface 模式同步：

1. **Enum 变体** `OldestDone { n: u32 }`（紧贴 TouchedYesterday）
2. **`name()` arm** → `"oldest_done"`
3. **`title()` arm** → 与 Recent / OldestN 同 N-only arm 集
4. **parser arm**：与 /recent / /oldest_n 同 N 处理（缺省 5，clamp 1..=20）
5. **en + zh registry** entries
6. **`ALL_HELP_TOPICS`** 加 `"oldest_done"`
7. **`format_help_for_topic("oldest_done")`** 详细文案（含四象限说明 +
   场景 + 与 /recent / /oldest_n 互补关系）
8. **`format_help_text`** 表格行
9. **两份 drift-defense test 列表**

#### 纯 formatter `format_oldest_done_reply`

与 `format_recent_reply` 同结构（同 line 格式 / 同 cap hint）— 让 owner
切换视角时心智一致；区别仅 `sort_by(asc)` + 标题用 `🪦 最早完成的 N`。

5 个 unit tests：parser（default / explicit / clamp 21→20 / 非数字
fallback）+ formatter（空集教学 / asc 排序 / 仅 done 过滤 / N cap +
余量 hint）。

### `src-tauri/src/telegram/bot.rs`

Handler 紧贴 Recent 之前（与 N-cap done 视图同 chunk）：

```rust
TgCommand::OldestDone { n } => {
    let views: Vec<TaskView> = read_tg_chat_task_views(chat_id.0);
    format_oldest_done_reply(&views, n)
}
```

## Key design decisions

- **format clone 不抽 generic**：与 `format_today_done_reply` /
  `format_yesterday_reply` 同既有 split 模板 — 单测点稳定 + 行内逻辑
  < 40 行，clone 维护成本可忽略
- **`🪦` emoji**：与 ✅（/recent）/ ⏳（/oldest_n）/ 🆕（/active_recent）
  对偶 — 「最老完成」语义；与 yesterday / today_done 的 📅 区分（那
  些按日期范围）
- **cap hint 含 cross-reference**：余量 hint 加 「或 /recent 看最近完成」
  — 让 owner 同时知道反向命令
- **复用 `/recent` 的 line 格式**：「YYYY-MM-DD HH:MM · title」与 /recent
  完全一致 — 心智复用 + 视觉切换无歧义
- **bug fixed: ASCII 直引号在 Rust 字面量内**：初版 help-detail 文案
  写了 `"最老/最新"`（ASCII `"..."`）在 Rust `"..."` 字面量内 — Rust
  parser 把内部 `"` 视作字符串结束 + `最新` 视作未知 prefix。改用
  fullwidth `「」` 修复（与 session summary 中 ASCII quote 教训一致）
- **5 个 unit tests pin 真实行为**（满足 GOAL.md "meaningful tests
  only"）：parser 4 路径（default / explicit / clamp / 非数字）+
  formatter 4 路径（empty / asc sort / done-only filter / N cap +
  余量 hint）

## Verification

- `cargo test --lib` — all 1633 tests pass（新 5 + 既有 1628）
- 三个 drift-defense test all pass
- 手测：
  - `/oldest_done` → 「🪦 最早完成的 5 条（共 N）」 + 列表 + cap hint
  - `/oldest_done 10` → 显 10 条
  - 无 done task → 友好教学指向 /done
  - `/oldest_done abc` → 退回 default 5
