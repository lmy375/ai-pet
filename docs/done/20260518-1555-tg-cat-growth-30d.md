# TG `/cat_growth_30d` 命令（iter #575）— /cat_growth_7d 30d cousin

## Background

完成 cat × period audit 四件套矩阵：

| period | growth | decay |
|--------|--------|-------|
| 7d     | /cat_growth_7d (iter #557) | /cat_decay_7d (iter #564) |
| 30d    | **/cat_growth_30d** (本 iter) | /cat_decay_30d (iter #572) |

owner 月度复盘想看「本月持续投入哪类知识」— 7d 信号太短噪音大；30d
平滑长周期 trend。

## Changes

6-surface 同步 + 共享算法 refactor（与 /cat_decay 30d 重构同 pattern）：

1. Enum `CatGrowth30d`（紧贴 `CatGrowth7d`）
2. name / parser / no-args title chain / registry en+zh / help-detail /
   help-table / 三份 drift lists

### Shared algorithm: `compute_cat_growth_rows(threshold_days)`

新 helper 在 bot.rs，参数化阈值天数。CatGrowth7d / CatGrowth30d 共调
（7 vs 30）— 解决两份近 30 行算法重复。镜像 `compute_cat_decay_rows`：
那是 max(updated_at) < cutoff stale 视角；本是 count(created_at >=
cutoff) growth 视角。

### Generic formatter: `format_cat_growth_reply(rows, threshold_days)`

旧 `format_cat_growth_7d_reply(rows)` 重构为参数化版本 — header 用
「近 N 天各类新增」+ 兜底「近 N 天各 cat 都没有 created」由参数注入。
旧 API 保留为 wrapper 兼容老测试。

## Key design decisions

- **共享算法精神延续**：本 iter 复刻 /cat_decay 30d 重构精神 — 阈值
  参数化让未来 cat_growth/decay_Nd 加变体 0 cost
- **同测试 case schema**：测「30d threshold 注入 header」+「30d 阈值
  注入 fallback」— 与 cat_decay 30d 测试镜像
- **`/cat_growth_30d` vs `/cat_decay_30d`**：前者列 30d 内 created
  ≥ 1 的活跃 cat（投入）；后者列 30d 内 updated 完全 0 的 stale cat
  （死气）。owner 月度复盘双用：「我做了啥（growth）+ 该清啥（decay）」
- **3 unit tests**：parser + 30d header + 30d fallback；共享 helper
  / formatter 已被既有 cat_growth_7d 测试覆盖
