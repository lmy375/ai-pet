# PanelTasks due date 颜色等级

## 需求

任务列表里 due 字段过去用 orange-fg / hardcoded `#ea580c` / 继承默认色三档，但前两档都是橙色调，扫一眼分不清紧迫度。TODO 要"过期红 / 24h 内黄 / 远期灰"，并把 chip 风格放到行内做色块标识，不靠悬停 tooltip。

## 实现

### 1. 主题加红色 tint

`src/theme.ts` 的 `ThemeTints` 加 `redBg / redFg`：

- light：`#fef2f2` / `#b91c1c`
- dark：`#2a1010` / `#fca5a5`

红色保留给"已经出问题 / 立刻看"的高紧迫信号；orange 仍是"warning 但不至于 fail" —— 语义不抢戏。

### 2. dueColor 改红/黄/灰，加 dueBg

`src/components/panel/PanelTasks.tsx`：

- `dueColor`：overdue → `var(--pet-tint-red-fg)`、soon → `var(--pet-tint-yellow-fg)`、normal → undefined（走默认色）
- 新增 `dueBg`：overdue → `var(--pet-tint-red-bg)`、soon → `var(--pet-tint-yellow-bg)`、normal → undefined

### 3. 渲染：urgent 走 chip 样式

`<span>` 加 background / padding `1px 6px` / `borderRadius: 999`，仅在非 normal 时启用。normal 行保持原平面文本，避免列表全是色块视觉爆炸。

## 验证

- `npx tsc --noEmit` clean
- 视觉：
  - 已过期任务：红字红底胶囊 chip，扫一眼锁定
  - 24h 内：黄字黄底胶囊 chip
  - 远期：默认 muted 文本
- light + dark 分别测：dark 下 red bg 是深红，fg 是粉红，文字对比够；yellow 同理

## 不在本轮范围

- 不在卡片整体边框上挂紧迫度颜色（现在只 due chip 着色）—— 边框红可能压过 focus ring，要做需要重新设计层叠优先级
- 剩 `/image -n` 多图生成
