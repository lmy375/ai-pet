# Modal 共享 overlay 组件（UI 美化 迭代 18）

## 背景

各 panel 散落十多个 `position:fixed inset:0 background:rgba(15,23,42,0.55) + center card` 组合。常见问题：
- overlay alpha 0.3 / 0.45 / 0.55 / 0.6 多种，视觉割裂
- shadow `0 20px 60px rgba(0,0,0,0.35)` 重复写 + dark 主题下不可见
- 没有统一的 Esc-to-close 监听（部分 dialog 仅 backdrop click，键盘不可达）
- 没有进入动画（pop-in / fade-in 各自实现 keyframes）

## 改动

### 新建 `components/panel/Modal.tsx`

API：
```tsx
<Modal open={dialogOpen} onClose={...} maxWidth={460} zIndex={100}>
  ...
</Modal>
```

特征：
- backdrop 用 `color-mix(<fg> 50%, transparent)` —— 浅 / 深主题下都能稳定形成"罩住"感
- card 用 `var(--pet-shadow-lg)` token，dark 主题自动加深
- pop-in 动画 (180ms) + fade-in (140ms)；prefers-reduced-motion 退化为无动画
- Esc 监听用 useEffect 全局注册，open=true 时挂；open=false 卸（避免后台残留）
- backdrop click 关闭；card stopPropagation 防穿透
- card 自带 maxHeight 85vh + overflow auto，长内容自然滚动

不做：
- 不渲染默认 close-✕（caller 自排，让 title 区灵活）
- 不引入 portal（Tauri 单 root，DOM 顺序无需调整）

### 迁移 2 个 dialog 作为 pilot

1. **PanelTasks** 手动标 done dialog（mark-done confirm）：
   - 原 47 行 backdrop + card + inline animation 模板 → `<Modal>` 包一层 + 内容保留
2. **PanelMemory** schedule edit dialog：
   - 同样迁移，zIndex 110（保留与 edit modal 100 的层级关系）

## 暂不动

- PanelTasks quickAdd modal：有自己的 pop-in keyframes + sticky header，layout 复杂，留下一轮
- PanelChat marks modal：有大量内部分组 / search input / batch ops，size 比简单 dialog 大几倍
- PanelMemory edit memory item dialog：另一种 layout（不同 padding / width 节奏）
- PanelDebug review modal、temp prompt modal：调试用，优先级低

这些都可在后续 iter 单独迁移；本轮只验证 pattern。

## 验收

- 在 PanelTasks 点 ✓ 标 done dialog：Esc 关 / backdrop 关 / 内容不变。
- 在 PanelMemory schedule edit dialog 同上。
- 两 dialog 都有 fade-in + pop-in 微动画（reduced-motion 退化）。
- `npx tsc --noEmit` 通过。
