# PanelMemory 编辑 modal 加 Cmd/Ctrl+S 保存（Iter R105）

> 对应需求（来自 docs/TODO.md）：
> PanelMemory 编辑 modal 加 Cmd/Ctrl+S 保存快捷键：当前必须点保存按钮，长描述场景下用快捷键更顺手（PanelTasks 详情面板已有此模式 line 2545，可仿照）。

## 目标

PanelMemory 编辑 modal 的 textarea 输 50+ 字描述时，要保存得鼠标离开 →
找到右下"保存"按钮 → 点击。中间一次手势切换。PanelTasks 详情面板的 detail.md
编辑 textarea 已用 `⌘S/Ctrl+S = 保存`（line 2565-）模式：textarea 内
按下时 preventDefault + 调用 save handler。

复用同款 pattern 接到 PanelMemory 的 description textarea。

## 非目标

- 不动 select / title input —— select 用 Enter 已是默认提交（无意义），
  title input 短不需要 keyboard shortcut；focus 在 textarea 时按 ⌘S 是
  最常见路径
- 不引入全局监听器 —— scoped 到 textarea onKeyDown 即可，避免与其它 panel
  的快捷键冲突
- 不动取消按钮 —— Esc 已经被 modal backdrop click 等效（但实际上 modal
  没监听 Esc 闭，留给后续 polish；本轮不扩展）

## 设计

### onKeyDown 加到 description textarea

```diff
 <textarea
   style={{ ...s.textarea, minHeight: editingItem.category === "butler_tasks" ? 100 : 60 }}
   maxLength={300}
   placeholder={CATEGORY_PLACEHOLDERS[editingItem.category] || ""}
   value={editingItem.description}
   onChange={(e) => setEditingItem({ ...editingItem, description: e.target.value })}
+  onKeyDown={(e) => {
+    // ⌘S/Ctrl+S 触发保存：与按钮等价。preventDefault 吃掉 webview 默认
+    // "另存为页面"行为；handleSaveEdit 内部已有 try/catch 防 race。
+    if ((e.metaKey || e.ctrlKey) && e.key.toLowerCase() === "s") {
+      e.preventDefault();
+      void handleSaveEdit();
+    }
+  }}
 />
```

### placeholder 提示

description textarea 的 placeholder 已通过 `CATEGORY_PLACEHOLDERS` 给出
具体范例。给保存按钮 title 补一句 "(⌘S 等价)"，让用户首次扫到入口知道
快捷键存在：

```diff
-<button style={s.btnPrimary} onClick={handleSaveEdit}>
+<button
+  style={s.btnPrimary}
+  onClick={handleSaveEdit}
+  title="保存到 memory index（⌘S/Ctrl+S 等价）"
+>
   保存
 </button>
```

### 测试

无单测；手测：
- modal 内 textarea focus 后按 ⌘S → 保存触发，modal 关闭
- description 空 / title 空 → handleSaveEdit 内部已有校验，按 ⌘S 走同路径
- 失败时已有 message 反馈
- focus 在 select / title input 时 ⌘S → 不触发（onKeyDown 只挂在 textarea）
  → 与现有"必须先 focus textarea"直觉一致；title 短不需要快捷键
