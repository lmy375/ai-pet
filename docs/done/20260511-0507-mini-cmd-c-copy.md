# 桌面气泡 ⌘+C 快捷复制最近 assistant 文本

## 需求

在桌面看完宠物刚回的一段话，想丢到飞书 / 钉钉，目前要：把鼠标移到 mini chat
气泡上 → hover 出"复制"按钮 → click → 切到飞书。⌘+C 是 OS 级肌肉记忆，让它
直接 copy 最近一条 assistant 文本能省一步并对齐用户预期。

TODO 标题里写的是 ⌘+回车，描述里是 ⌘+C；按 OS-标准 copy 快捷键实现。

## 设计

只在 selection **为空**时拦截 ⌘+C：

- 用户精确选了一段文字 → 让浏览器原生复制选词，行为照旧
- 用户什么都没选 → 拦截 ⌘+C，复制最近 assistant 消息文本

这样两条路径都自然：精确复制 vs "我刚看完它说啥就拿这段"。

## 实现

`src/components/ChatMini.tsx`：

- 加 `copyToast: "none" | "done" | "err"` state
- useEffect 在 visible 期间挂 `window.addEventListener("keydown", handler)`：
  - 检 metaKey / ctrlKey + key === "c"
  - 检 `window.getSelection().toString()` 非空 → 不拦截
  - 找 visibleItems 里最近 assistant message
  - extractText 取文本 → navigator.clipboard.writeText
  - setCopyToast 反馈，1.5s 自清
- 渲染 fixed top:24 居中浮的小药丸 toast，pointerEvents none，1.5s 自动消失
- visible=false（panel 切走）时移除监听器，避免后台抢 ⌘+C

## 验证

- `npx tsc --noEmit` clean
- 行为：
  - 桌面看到宠物回了"今天天气不错~" → ⌘+C → 顶部药丸"✓ 已复制最近回复" → 飞书粘贴出"今天天气不错~"
  - 鼠标在某条历史里选了"天气不错" → ⌘+C → 浏览器原生复制"天气不错"，不走拦截
  - 桌面气泡隐藏（ChatMini visible=false） → ⌘+C 不被拦
  - 用户在 ChatPanel textarea 里输入但什么都没选 → ⌘+C 走拦截（一致体感"⌘+C 在桌面就是要最近回复"）
