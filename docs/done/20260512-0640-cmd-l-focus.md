# ChatMini ⌘L 全局聚焦输入框

## 需求

桌面 pet 窗口当前需要点 ChatPanel textarea 才能输入。用户从 Live2D 区 /
ChatMini 历史区 hover 完想立刻回去敲新一条得手动点。加 ⌘L 全局快捷把焦
点送回 textarea，类似 terminal cmd+L 的"快速重置注意力"反射。

## 实现

`src/components/ChatPanel.tsx`：

- 新 useEffect 挂 window-level `keydown` 监听：
  - 守门：metaKey || ctrlKey 必须按下；key === "l" 或 "L"；alt / shift
    不能按（避免与未来其它快捷冲突）
  - `e.preventDefault()` 防偶发默认（浏览器 ⌘L 是 focus address bar，
    Tauri WKWebView 没地址栏所以一般无影响，但保险起见）
  - `textareaRef.current.focus()` + `setSelectionRange(len, len)` 把光标
    移到末尾，已有内容时也能直接续写
  - setSelectionRange try/catch 包：极端 input 类型不支持时静默退化

## 验证

- `npx tsc --noEmit` clean
- 行为：
  - 鼠标在 Live2D 区 → 按 ⌘L → textarea 立刻聚焦，光标在末尾
  - input 已有 "hello" → ⌘L → 焦点回 input，光标在 "hello|" 后面
  - 同时按 ⌘+Shift+L / ⌘+Alt+L → 守门拒绝，不影响其它快捷
  - 浏览器原生 ⌘L "address bar focus" 行为被吃掉（Tauri 本就没地址栏，
    无感）
  - panel 窗口 / debug 窗口未挂这个 listener（ChatPanel 只渲染在 pet
    窗口），不影响那边
  - PanelChat 自己的 textarea 有独立 keyboard 路径，⌘L 不会被这里抢
    （panel webview 不加载 ChatPanel 组件）

## 不在本轮范围

- 没做 ⌘K / ⌘/ 等其它常见 IDE-类快捷绑定：那是 GOAL.md 待确认级别的
  键盘体系，单独需求
- 没让 ⌘L 顺带清空 input（terminal cmd+L 真语义是清屏）：用户提交后
  Enter 已自清；强清会丢未发草稿
- 没在 panel chat textarea 加同款：panel 已有 ⌘F / ⌘+C 等快捷；后续可
  作单独需求统一
