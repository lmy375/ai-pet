# PanelSettings 主题 accent 自定义

## 需求

`--pet-color-accent` 是主品牌色，控制 active tab / primary button / 高亮
链接 / user bubble 等几十处。当前只有 sky 蓝写死，用户没法挑自己喜欢的
色调。加 5 色 palette（蓝 / 绿 / 紫 / 橙 / 玫红）让 advanced 用户切，跨
窗口同步生效。

## 实现

### `src/theme.ts`

- 新 `Accent` 类型：`"default" | "green" | "purple" | "orange" | "rose"`
  （default 沿用现有 sky 0ea5e9 / 38bdf8 兼容老用户）
- 新 `ACCENT_VALUES: Record<Accent, Record<Theme, string>>`：5 色 × 2 主题
  共 10 个 hex；dark 普遍提亮一档保对比度
- 新 `ACCENT_OPTIONS` UI 元数据数组（label + swatch hex）给设置页选项用
- `applyTheme(theme, accent?)` 加可选 accent 参数；不传 / default 走原
  token；传则覆盖 `--pet-color-accent` 字段。`data-accent` 写到 root attr
- 新 `getStoredAccent / setStoredAccent`：localStorage key `pet-accent`，
  与既有 theme storage 模式一致

### `src/App.tsx`

- 顶层 bootstrap：`applyTheme(getStoredTheme(), getStoredAccent())`
- 既有 theme-change listener 加 `getStoredAccent()` 第二参；新加
  accent-change listener，payload 校验白名单防注入

### `src/PanelApp.tsx`

- useState initializer 改 `applyTheme(t, getStoredAccent())`
- `toggleTheme` 同样带上 accent，否则切深浅会丢自定义色
- listener effect 拆 pTheme + pAccent 两条 listen，cleanup 各自 unlisten

### `src/components/panel/PanelSettings.tsx`

- 新 `AccentPickerRow` 组件：5 个 28px 圆形 swatch 按钮（active 加白边 + ✓
  标记，inactive 透明边）+ "立即同步到桌面 / 调试窗口" 提示
- 与 ThemeToggleRow 同模式（dynamic import theme 模块 + emit 事件）
- 外观 section 内：`<ThemeToggleRow />` 下方 marginTop:12 加 `<AccentPickerRow />`

## 为什么不写 config.yaml

theme / accent 是 UI 偏好（每台机器、每个用户独立），不该和 LLM 行为
参数（model / api_key / motion mapping）混在 config.yaml 里：
- localStorage 已是既有 theme 的存档处，accent 跟着走最自然
- 用户跨设备会自动 reset 到 default —— 与"不做云同步"决策一致
- 不需扩 Rust Config struct + 迁移老 config.yaml 兼容

## 验证

- `npx tsc --noEmit` clean
- 行为：
  - 打开 PanelSettings 外观区 → 主题切换下方出现 Accent 5 圆 + "立即同步" 提示
  - 选 "绿" → 整个 panel 的 active tab 下划线 / "+ 新会话" 按钮 / user
    气泡 / primary action 立刻变绿；桌面 pet 窗口 ChatMini 也跟着变
  - 切深色 → 绿色提亮版（emerald-400），对比度仍可读
  - 重启 panel → 绿色保留（localStorage 已写）
  - 选 "蓝" → 回 default
  - 调试窗口暂不监听（独立 webview 用 hardcoded 色），不影响主流程

## 不在本轮范围

- 没让 DebugApp 跟主题 / accent：它有自己的 hardcoded #0ea5e9 sky 链路，
  迁移到 var(--pet-color-accent) 是单独的工作量（与本次 accent 扩展正交）
- 没做"自定义 hex"输入：advanced 用户能改 5 个预设已足够，自定义 hex 涉
  及 a11y 对比度校验 + dark 配对自动算法，单独的产品决策
- 没做配色对其它 tint 的连带调整（如 user bubble background）：accent 仅
  作"主品牌"使用，与 6 对 section tint 各司其职，本轮专注 accent
