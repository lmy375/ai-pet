# PanelChat "📌 全部标记消息" modal

## 需求

iter #220 给消息加了 📌 按钮（localStorage 收藏）。但 follow-up "查
看所有标记"页面没做 —— 用户标完就找不到了。本 iter 补 modal：从
markedMessages set 反查每个 (sessionId, itemIdx) 对应的消息内容，
渲列表，点击跳源。

## 实现

`src/components/panel/PanelChat.tsx`：

### state + loader

- 新 type `MarkedEntry = {sessionId, sessionTitle, itemIdx, role, content}`
- 新 state：
  - `marksModalOpen: boolean`
  - `marksModalEntries: MarkedEntry[] | null`（null = loading 中）
- `openMarksModal()` async：
  - 按 sessionId 分组 markedMessages keys
  - 对每个 sessionId batch `invoke<Session>("load_session", {id})`
  - walk `session.items[idx]` 收集 entry（skip non user/assistant 防 dangling）
  - 单 session 失败 swallow，其它 session 仍能显
- entries 建好后 setMarksModalEntries

### 触发按钮

session bar 工具栏的 🔍 搜索按钮旁新增一个 "📌 N" 按钮：
- 仅 markedMessages.size > 0 时显
- yellow tint 配色让 pinned 信号显眼
- 点击 → `setShowSessionList(false)` + `openMarksModal()`

### Modal

fixed overlay 居中 640px / 75vh max：
- header：`📌 全部标记消息 (N)` + 关闭 ✕
- body：分三种状态渲染
  - `null` → "加载中…" 占位
  - 空数组 → "（没有可显示的标记 — session 可能已删 / 消息可能已被改）"
  - 有 entries → 列表，每条：
    - 上行 meta：role 🧑/🐾 + 会话标题 (ellipsis) + #N（itemIdx+1）
    - 下行内容：4 行 webkit-line-clamp 截断 + pre-wrap
    - 整行可点 → 关 modal + loadSession（若需切换） + setPendingScroll(itemIdx)
- backdrop / ✕ / Esc 都关
- 跳转复用既有 pendingScroll + highlight 1.5s 路径（与跨会话搜索同款）

## 验证

- `npx tsc --noEmit`：clean
- 行为：
  - 标过任意消息 → session bar 上浮 📌 N 按钮
  - 点击 → modal 显加载中 → 异步拉完显列表
  - 点列表某条 → 跳到该 session + 滚到该 item + 1.5s 黄色高亮
  - 跳完后 modal 关
  - 全部 dangling（标过的 session 都删了）→ modal 显空提示
  - 跨 session 标记 → modal 内不同 session entries 混在一起，按 set
    遍历序展示
  - 标记数 0 → 按钮不显，无入口（与 iter #220 一致）

## 不在本轮范围

- 没做"取消标记"行内按钮：modal 内列表是阅读视图；取消标记走原消息
  上的 📌 按钮
- 没做"分组按 session"展示（折叠 session header）：当前 < 50 条规模
  扁平列表足够；分组让 UI 重
- 没做"按时间排序"（最新标在上）：标记时间没记，只有标记键存在；
  当前迭代顺序与 Set insertion 一致，足够直觉
- 没在 markedMessages 改变后实时刷 modal（用户在 modal 开着时另一窗
  口标了消息）：关闭后重开即刷新；单 panel 内不需要 reactive
- 没让搜索结果与 marks 共一 list（unified history view）：search 是
  动态查询，marks 是静态收藏，两者 UX 不同
