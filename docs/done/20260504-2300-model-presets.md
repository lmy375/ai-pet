# 模型下拉预设 — 开发计划

> 对应需求（来自 docs/TODO.md「已确认」）：
> 模型下拉预设：settings 的 model 字段现在是裸 input，加常见模型下拉（gpt-4o-mini / gpt-4o / claude-sonnet-4-6 等）减少打字。

## 目标

在「设置」标签页的 LLM 配置区，把 `model` 字段从裸 `<input>` 升级为带常见模型建议的 `<input list="...">` + `<datalist>`。用户聚焦时下拉显示常见模型名，但仍可手输任意值（兼容 OpenAI / Anthropic / DeepSeek / 自托管 / OpenRouter 等所有 OpenAI-compatible 后端）。

价值：
- 消除常见模型名的拼写错误（典型 footgun：`gpt4o-mini` / `gpt-4o-mini`）
- 提示用户已知可用的最新模型，无需开浏览器查
- 完全向后兼容，零破坏

## 非目标

- 不改 backend `AppSettings.model` 字段（仍是 String）
- 不做"测试连接"按钮 —— 用户已经可以发一条聊天 fail-fast
- 不做"按 api_base 自动建议匹配模型"—— 复杂度高、收益不明显
- 不写到 README —— UX 改进非亮点级

## 设计

### 模型清单

按提供商分组，每组挑当前 representative 几个：

```
OpenAI:    gpt-4o, gpt-4o-mini, gpt-4-turbo, o3-mini
Anthropic: claude-opus-4-7, claude-sonnet-4-6, claude-haiku-4-5
DeepSeek:  deepseek-chat, deepseek-reasoner
其它:      gemini-2.0-flash, qwen2.5-72b-instruct
```

放进 `MODEL_PRESETS: { value: string; label?: string }[]`。`label` 可选，用于显示分组提示（如 "OpenAI / GPT-4o mini"）。

### UI

```tsx
<input
  list="model-presets"
  type="text"
  value={form.model}
  onChange={...}
/>
<datalist id="model-presets">
  {MODEL_PRESETS.map(({ value, label }) => (
    <option key={value} value={value}>{label ?? value}</option>
  ))}
</datalist>
```

datalist 渲染由浏览器原生处理，不需要任何状态机；用户键入时自动 fuzzy 匹配并展示建议；点 datalist 选项 = 直接填入对应 value。Edge case 全部由 input 的现有 onChange 兜底。

### 变更范围

仅 `PanelSettings.tsx`：原 LLM 配置区里 `model` 那行 input 替换为 datalist 形态 + 同文件加 `MODEL_PRESETS` 常量。无 hooks 改动、无 backend 改动。
