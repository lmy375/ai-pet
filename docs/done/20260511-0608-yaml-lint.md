# 设置页 raw YAML 加 lint

## 需求

YAML 写错时只有点"保存"才看到 backend 报错；输错的 schema（image_size 写漏个引
号、proactive 缩进吃了 tab）只能"先保存 → 看后端 → 改 → 再保存"循环验证。
实时 lint 让用户敲到一半就看见错误 + 行号，闭环更短。

## 设计

不引第三方 yaml 库（js-yaml ~30KB bundled）。复用后端已有 `serde_yaml::from_str
::<AppSettings>` 路径 —— 新 `validate_config_raw(content) -> Result<(), String>`
命令做同样的解析但不落盘。前端 debounce 500ms 调一次，错误直接显在 textarea
下方。

好处：lint 与真实 save 走同一份解析逻辑（serde_yaml + AppSettings schema），
不会出现"lint 过但保存又拒"的差异；同时 schema 校验也免费（不光语法，连
"陌生字段 / 类型错"也能拦）。

## 实现

### 后端

`src-tauri/src/commands/settings.rs`：

```rust
#[tauri::command]
pub fn validate_config_raw(content: String) -> Result<(), String> {
    if content.trim().is_empty() {
        return Ok(()); // 清空中是中间态，不报错
    }
    let _: AppSettings = serde_yaml::from_str(&content)
        .map_err(|e| format!("YAML 解析失败: {}", e))?;
    Ok(())
}
```

`src-tauri/src/lib.rs` 注册 `commands::settings::validate_config_raw`。

### 前端

`src/components/panel/PanelSettings.tsx`：

- `yamlError: string | null` state，三态：
  - `null` = 检查中 / typing 期间，UI 不渲染反馈（避免抖）
  - `""` = 合法
  - 非空字符串 = serde_yaml 错误透传
- useEffect on `[rawYaml, viewMode, loaded]`：
  - viewMode 非 raw / 未 loaded → 不跑
  - setYamlError(null) 进入"检查中"
  - 500ms setTimeout 后 invoke validate_config_raw，写结果
  - 清理 timer 防上一次 typed 的请求覆盖最新
- 渲染：
  - textarea borderColor 错时换红（tint-red-fg）
  - textarea 下方报错栏：红底红字 + ⚠ + 多行 wrap，错误透传文案（含 `at line X column Y` 等 serde_yaml 自带定位）
  - 合法时小绿"✓ YAML 合法"
- 保存按钮 disable：仅在 `yamlError` 非 "" 且非 null 时禁用（pending 不禁用，让用户敲完保存仍能走老路径）

## 验证

- `npx tsc --noEmit` clean
- `cargo check` clean
- 行为：
  - 输 `api_key: xxx` 后中间删个引号 → ~500ms 后红框 + 错误带行号
  - 输入合法 → 绿色 "✓ YAML 合法"
  - typing 半截（中间态）→ 不闪烁红绿（null 期间空态）
  - 错状态点保存 → 按钮 disabled，tooltip 提示
  - viewMode 切到 form → 不再 validate（与 raw mode 解耦）

## 不在本轮范围

- 没把错误行号映射到 textarea 内 highlight：serde_yaml 错误的 line/column 是
  文本里的位置，要 highlight 需算 textarea 内的字符偏移 + setSelectionRange。
  下版本可以扩 —— 当前先把文案显出来用户就能 ⌘F 找到
- 没把 lint 暴露给 form 模式：form 输入由 React 各 onChange 收敛类型，schema
  本身就强，没"YAML 出错"路径
