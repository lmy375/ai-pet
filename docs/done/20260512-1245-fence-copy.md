# Fence code block 一键复制

## 需求

上一轮加的 fence code block 渲染只有视觉，不带交互。LLM 输出代码 / 命令
时用户想复制还得手选 + ⌘C。给 fence 块加 hover-only 右上角 📋 复制按钮 +
"✓"反馈。

## 实现

`src/utils/inlineMarkdown.tsx`：

- import `useState`（之前是纯 `import type { ReactNode }`）
- parseMarkdown 的 fence 渲染分支改成 `<FenceCodeBlock lang={...} code={...} />`
  组件实例 —— useState 不能放在 parseMarkdown 的 for-loop 内（hooks 顺序
  规则），必须独立组件
- 新 `FenceCodeBlock` 组件：
  - `hovered / copied` state
  - `onMouseEnter / Leave` 控制 hover
  - `handleCopy` → `navigator.clipboard.writeText(code)` + 1.5s "✓"
  - 复制按钮渲染：`hovered || copied` 时显（copied 状态强制可见 1.5s）
  - 视觉：右上角 absolute，nav 蓝边 / 暖琥珀底 / 10px 文字
- `CODE_LANG_BADGE_STYLE.right` → `.left`：lang badge 挪到左上角，给右上
  角让位复制按钮（避免色块挤在同一角）
- 新 `CODE_COPY_BTN_STYLE` const，配色与代码块底协调

## 验证

- `npx tsc --noEmit` clean
- 行为：
  - hover fence code 块 → 右上角浮 📋 按钮，左上角 lang badge
  - 点 📋 → 写剪贴板成功 → 按钮变绿 ✓ 1.5s → 回 📋
  - 鼠标移出 → 按钮淡出（除非 copied 状态强制留显）
  - 触发于 ChatMini / PanelChat 历史中所有 fence code（共用 parseMarkdown）
  - 无 lang 的 fence ` ``` \n... \n``` ` → 左上角不显 lang badge，仅右上
    角 copy 按钮
  - 多个 fence 块共存 → 各自独立的 hover / copied 状态

## 不在本轮范围

- 没做"按行号复制"或"复制一段范围"：单 fence 复制已经覆盖 95% 用例
- 没改 inline `<code>` 加复制按钮：单 token 复制不值得；用户直接选中即可
- 没在 panel 任务 detail.md preview 内的代码块加 hover 复制：那走的不是
  parseMarkdown 路径（detail preview 是 plain text）；后续若 detail 支持
  markdown 渲染再继承
