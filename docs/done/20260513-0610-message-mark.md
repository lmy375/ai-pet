# PanelChat 消息 "📌 标记" 按钮

## 需求

聊天里偶尔有"这条得收藏一下回头看"的消息（LLM 写的好建议 / 用户自
己琢磨出的方法）—— 当前只能截图 / 手抄。给一个 📌 按钮 toggle 标记，
localStorage 持久化，跨重启保留。本轮先做"按钮 + 标记 + 持久"三件
事；"查看全部标记"页面留作 follow-up。

## 实现

### PanelChat (state + 持久化)

`src/components/panel/PanelChat.tsx`：

- 新 state `markedMessages: Set<string>`，localStorage key
  `pet-chat-marked-messages`，挂载读
- `toggleMessageMark(key)` callback：toggle set + 写盘
- 两条 CopyableMessage 路径（user / assistant）都传：
  - `marked={markedMessages.has(`${sessionId}::${i}`)}`
  - `onToggleMark={() => toggleMessageMark(`${sessionId}::${i}`)}`
- key 用 `(sessionId, itemIdx)` 元组拼字符串 —— 上游消息编辑 / 删除
  时索引可能漂移（dangling key 不要紧，用户重 mark 即更新）；hash
  content 太重不值

### panelChatBits CopyableMessage

`src/components/panel/panelChatBits.tsx`：

- 新增 props `marked?: boolean` + `onToggleMark?: () => void`
- 新 inline `markButton`：未传 onToggleMark → return null（tool / error
  消息无 mark 语义）
- 视觉：
  - default outline + muted 灰（同复制按钮 default 视觉）
  - marked → yellow tint bg + yellow tint fg + bold + force opacity 1
    （与 copied 状态同语义"持久态"，从 hover-only 升到 always-visible）
  - emoji 📌 一个字符压紧
- 位置：
  - user 行：`[mark][copy][bubble]`（左侧）
  - assistant 行：`[bubble][reactionRow][copy][mark]`（最右）
- 共用 .pet-copy-btn class → hover-only fade 行为自动继承

## 验证

- `npx tsc --noEmit`：clean
- 行为：
  - 任意 chat 消息 hover → 浮 📌 按钮（与既有复制 / reaction 同 hover-fade）
  - 点击 → 黄底持久显示 + 写入 markedMessages set
  - 再点击 → 取消 mark，回 outline 灰
  - 重启 panel → markedMessages 还原，黄底自动恢复
  - tool / error 行不渲染（无 onToggleMark prop）
  - 跨 session 隔离：同 itemIdx 在不同 sessionId 是独立 key
  - localStorage 损坏 / 私密模式 → 空 set 退化，不阻塞 chat

## 不在本轮范围

- "看全部标记"页面：另一个 panel / modal / sidebar item，需要 backend
  按 marked keys 拉对应消息内容；scope 翻倍，留 follow-up
- 没用 content hash 做 key（防消息删 / 编辑后 dangling）：上游 chat
  消息很少删 / 改，dangling 概率低；hash 也无法 100% 防止（编辑后 hash
  变化才是问题，等于"mark 跟着内容"，可能更糟）
- 没做"批量取消所有标记"按钮：用户单条取消是基线；批量页面同 follow-up
- 没集成 PanelDebug issue 模板（带上 marked 消息）：那个模板已经覆盖 turn 上下文；
  marked 消息是用户主观收藏，不是 issue 关键信息
