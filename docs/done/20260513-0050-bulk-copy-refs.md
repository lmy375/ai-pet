# PanelTasks bulk "🔗 拼为 ref" 操作

## 需求

iter #189 / #203 已经支持单条任务复制为 ref（`「title」`）。bulk select
有"复制标题"和"复制为 MD"两个 copy 动作，但没"拼成 ref 列表"——用户
想在 chat 里一句话引用多条任务（"`「A」「B」「C」都要在周五前清完`"）
没快路径。补 bulk action。

## 实现

`src/components/panel/PanelTasks.tsx`：

- 新 `handleBulkCopyAsRefs` callback：
  - 从 selected Set 取出 tasks 全集中的对应行（order 与 visibleTasks
    一致）
  - 每条 title 包成 `「title」`，全部 join 成单行字符串
    （`「A」「B」「C」`，无中间分隔 —— 全角直角引号自带视觉边界，紧
    凑度更高）
  - 写剪贴板 + `bulkResultMsg` 4s toast 反馈
- bulk action 行中"复制标题"和"复制为 MD"之间插入按钮 "🔗 拼为 ref"
- title tooltip：解释粘到 chat 后每 token 都是 hover-able ref（hover
  显状态 / 双击跳源）

## 验证

- `npx tsc --noEmit`：clean
- 行为：
  - 多选 3 条任务（A / B / C）→ 点 "🔗 拼为 ref"
  - 剪贴板 = `「A」「B」「C」`
  - 粘到 chat → 三个 dotted underline ref token 紧邻，每个可独立 hover /
    双击
  - 选中条全被外部状态清掉（选完又 reload）→ "无可复制任务"反馈
  - 4s 自清反馈

## 不在本轮范围

- 没做"拼 ref 列表带分隔符（如 `\n` / `, ` 可选）"：紧凑形态最常用；
  需要时用户自己空格分隔
- 没做"拼为带状态的 ref 列表"（`「A」(pending) 「B」(done)`）：状态
  在 hover preview 已可见；附在文本里反而冗余
- 没把这条 bulk 动作转用 ⌘K picker 的 multi-select 模式：picker 当前
  是单选；多选改动会扩 scope
- 没改 PanelMemory 同款 bulk 动作（PanelMemory 没 multi-select）：单
  独 TODO 项若有需求
