# PanelDebug "🔄 reload" 按钮

## 需求

用户修改外部文件（手 vim config.yaml / 改 SOUL.md / 替换 memory 索
引）后，DebugApp 内多个 useEffect 拉的数据是 staleness。当前没"强
制刷新"入口，得关 DebugApp 重开。给一个 reload 按钮直接走
window.location.reload()。

## 实现

### DebugApp（持久 activeTab）

`src/DebugApp.tsx`：

- `activeTab` 初始化逻辑加 sessionStorage hop：read
  `pet-debug-reload-tab` 一次性消费 + 类型守卫，未命中走默认"应用"
- reload 前后 tab 保持 —— 避免 reload 后用户被甩回"应用" tab 丢上下文

### PanelDebug 按钮

`src/components/panel/PanelDebug.tsx`：

- 在 "🧹 重置 stash" 按钮后加 "🔄 reload" 按钮
- onClick：写 sessionStorage["pet-debug-reload-tab"] = "应用"
  （PanelDebug 必然在"应用" tab 才看到这按钮，写固定值即可）+ 调
  `window.location.reload()`
- accent 蓝底白字与其它 toolbar 按钮区分（reload 是"全量重启"语义）
- title tooltip 解释路径与 stash 重置的差异（前端 vs 后端 axis）

## 验证

- `npx tsc --noEmit`：clean
- 行为：
  - 在 vim 改 ~/.config/pet/config.yaml → DebugApp 内点 🔄 reload
  - 窗口刷新（所有 useEffect 重跑）→ panel 显新数据
  - reload 后用户落回"应用" tab（与点击时一致）
  - 与 🧹 重置 stash 共存：前者清前端 state + reload，后者清后端
    in-process stash（PanelDebug audit 行 / prompt modal 等）

## 不在本轮范围

- 没做 reload 前的"未保存提醒"：PanelDebug 是只读 view（除工具风险
  toggle 等），用户主动触发 reload 不会丢编辑内容
- 没做"刷哪段" 细粒度（如仅 memory）：full reload 简单可靠；细粒度
  需要每个 useEffect 暴露 manual refresher，scope 翻倍
- 没在 PanelApp / DebugApp 其它 tab 同款按钮：DebugApp 内"应用" tab
  数据最复杂；"日志" / "统计" tab 各自简单或已有 refresh 路径
- 没 hop activeTab 到 DebugApp 当前真值（写死 "应用"）：用户从
  PanelDebug 内点击时必然在"应用" tab；其它 tab 没此按钮，不会跑此
  代码路径
