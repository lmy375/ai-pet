# TG `/here_pin` 命令（iter #591）— pinned → transient_note pipeline

## Background

owner 早会前 / sprint start 想让 pet 「这是我此刻关注什么」明示 — 让
pet 主动 reply 时围绕 pinned task。当前 path：

1. `/pinned` 看 list
2. 手抄 titles 到 clipboard
3. `/transient 「整理 Downloads」「写周报」...` 注入

3 步麻烦。本 iter 加 `/here_pin` 一键 pipeline。

## Changes

6-surface 同步：

1. Enum `HerePin`（紧贴 `CatTop`）
2. name / parser / no-args title chain / registry en+zh / help-detail /
   help-table / 三份 drift lists
3. 新 pure `format_here_pin_reply(titles, until_local)` formatter
4. Handler scan chat-scoped views filter pinned → 拼紧凑 pin context
   字串 → set_transient_note 60min → 拿 until_iso → formatter
5. /help_table pin 家族 + family detail map 同步加入 /here_pin

## Output

```
📌 已注入 N 条 pinned task 到 transient_note（到 HH:MM 失效）
· 「整理 Downloads」
· 「写周报」
· 「review PR」
```

空 → 「无 pinned task — 试 /pin <title> 钉一条」+ 教学指 /pinned。

## Key design decisions

- **transient_note text 紧凑单行**：handler 拼「📌 当前 pin context：
  「t1」「t2」...」 — pet 系统 prompt 注入时 token 经济。多 title 用
  「」全角引号包裹无空格 — 比 `「t1」, 「t2」` 节字符
- **60min hard-coded**：常用 sprint kickoff timing；不暴露参数避增表
  面积。owner 想自定义时间用 `/transient <text> N` 走自由 path
- **reuse `set_transient_note`**：与 /transient 同后端 — 不引入新
  storage 路径
- **空 pinned tooltip 教学指 /pin**：避免 owner 困惑「我没钉怎么用」
- **chat-scoped pinned**：read_tg_chat_task_views 已 filter origin —
  不跨 chat 污染
- **保 views.compare_for_queue 既有顺序**：title 顺序与 /pinned 输出
  一致，让 owner 视觉 anchor 不变
- **4 unit tests**：parser + empty fallback + 多 title render + None
  until fallback (`到 ? 失效`)
