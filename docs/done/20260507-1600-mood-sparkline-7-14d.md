# mood sparkline 7→14 天扩窗 — 开发计划

> 对应需求（来自 docs/TODO.md）：
> mood sparkline 7→14 天扩窗：现固定 7 天，复盘"两周心情走向"看不到；加 chip "7d / 14d" 切窗口长度，复用既有 days 参数。

## 目标

PanelPersona 的 mood sparkline 当前固定 7 天。后端 `get_mood_daily_motions(days)`
和 `get_mood_half_day_motions(days)` 已支持任意窗口；只需前端切换。本轮加
"7d / 14d" 切换 chip，让用户复盘"近两周心情走向"能看到更长 trend。

## 非目标

- 不做 30d / 90d / custom —— 14d 已覆盖周回顾；更长窗口的 sparkline 在
  柱宽紧凑下视觉信息密度低，加档反而让用户犹豫。
- 不持久化窗口选择 —— 与既有 sparkline 切换型 state 一致（临时视角）。
- 不动 `clearDays` 默认 (7) —— 那是清理 mood_history 的入口，与 sparkline
  视图不直接挂钩。

## 设计

### state 位置

`sparklineDays: 7 | 14` default 7，放在 PanelPersona（父级），因为
fetchAll 在父级，需要 days 作为 useEffect 依赖项。

### useEffect 依赖

`fetchAll` 的 useEffect 依赖 `[]` 改为 `[sparklineDays]`，让窗口切换时立
即重新 fetch daily。post-clear 的 daily refetch 也跟这个 days。

### MoodSparkline props

加 `windowDays: number` + `onWindowDaysChange: (n: number) => void`：
- header 的 chip 行末尾插 `7d` / `14d` 互斥 chip，与既有 motion chip /
  早晚分段 chip 同视觉规格
- half-day fetch effect 改为依赖 windowDays 也跟着切

### chip 样式

复用既有 motion-filter chip 的"selected 填充 + inactive 描边"配色。这里
selected 用 slate（`#cbd5e1` bg / `#475569` fg），中性色——窗口长度本身
不携带情绪语义。

- 既有 sparkline header chip 行容器
