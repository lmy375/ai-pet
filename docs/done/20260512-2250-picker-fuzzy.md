# PanelChat ⌘K task picker char-order 子序列 fuzzy 匹配

## 需求

iter #178 引入 ⌘K picker 时的过滤是 `title.toLowerCase().includes(q)`。
includes 要求 query 是 title 的连续子串 —— 用户敲 "intDown" 想找"整
理 Downloads" 命中不了。char-order subsequence 匹配（query 每个字符
按顺序在 target 出现）更容错，是 fuzzy match 最朴素实现。

## 实现

`src/components/panel/PanelChat.tsx` 替换 picker filter IIFE：

- 新 inline helper `fuzzyMatch(query, target) => score | null`：
  - 单遍扫 target，对每个 query char 找下一处出现位置
  - 全找到 → 返 `(lastMatch - firstMatch + 1) * 100 + firstMatch`
    （越小越优；紧凑度权重高于位置）
  - 找不全 → null（不命中）
  - 空 query → score 0（直接列全集，调用方早 return）
- filter 改成 `(t) => fuzzyMatch(q, t.title.toLowerCase()) !== null`
  并按 score 升序排
- 空 query 走 fast-path 直接返 taskPickerTasks 原序，避免无意义打分

picker placeholder 改成"按标题模糊搜任务…（按字符顺序匹配；↑↓ 选 /
Enter 插入 / Esc 关闭）"——告诉用户输入语义变了。

## 验证

- `npx tsc --noEmit`：clean
- 行为：
  - 空 query → 全集（与原一致）
  - 敲 "down" → 命中 "整理 Downloads"（substring 路径也命中，无回退）
  - 敲 "intDown" → 命中 "整理 Downloads"（fuzzy 路径，substring 不会
    命中）
  - 敲 "ID" → 命中所有含 i...d 的 title（紧凑度 sort 让最匹配的靠前）
  - 敲 "xyz"（target 里无 x/y/z）→ 无命中
  - 多任务都命中 → 按 score 排序，紧凑匹配在前

排序 tiebreak 用 firstMatch（位置靠前的优先），让用户敲首字符时第
一条命中是最直觉的"前缀类"。

## 不在本轮范围

- 没做"camelCase 关键字加权"（如 "intDown" 里 D 应该是词边界 = bonus）：
  现 score 已经能让最佳匹配上浮；加权重权重需要 word-boundary 检测
- 没做"高亮匹配字符"（fzf 风格 underline）：picker 行已经有 status pill
  + title，视觉密度饱和；高亮逻辑会让单行渲染翻倍
- 没改 PanelTasks 的 search input（也是 includes 路径）：当前 search
  用全文匹配 title + body，fuzzy 在全文场景下会过宽（容易误命中），
  保留严格 includes 更稳
- 没做 trigram / Levenshtein 等更复杂的算法：char-order 子序列已覆盖
  常见 typo 缺漏字符；replacement / transposition 的场景边际
