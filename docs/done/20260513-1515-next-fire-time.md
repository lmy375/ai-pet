# PanelMemory butler_tasks "⏰ 下次触发" 相对时间

## 需求

butler_tasks item 行已有 schedule chip（🔁 每天 09:00 / 📅 once /
⏳ deadline）显绝对时间。但用户扫读时想知"这条还多久会跑"—— 得心
算。补 inline 相对时间字段。

## 实现

`src/components/panel/PanelMemory.tsx` 在 ✏️ 改 schedule 按钮后插入
小灰字 span（仅 butler_tasks + parsed schedule 时浮）：

- 计算 target Date：
  - every → 今天的 HH:MM；若已过 → 明天的 HH:MM
  - once / deadline → 绝对 yyyy-mm-dd HH:MM
- diff = target - now
- 相对时间档：
  - past < 1m → "刚过"
  - past < 1h → "已过 N 分"
  - past < 1d → "已过 N 时"
  - past ≥ 1d → "已过 N 天"
  - 未来 < 1m → "1 分内"
  - 未来 < 1h → "N 分后"
  - 未来 < 1d → "N 时后"
  - 未来 ≥ 1d → "N 天后"
- past 时走 orange tint fg 标注未触发的过期；未来走 muted
- title tooltip 显完整绝对时刻

## 验证

- `npx tsc --noEmit`：clean
- 行为：
  - 任务 `[every: 09:00]` 当前是 8:30 → "⏰ 30 分后"
  - 同任务 10:00 → "⏰ 23 时后"（next 是明天 9:00）
  - `[once: 2026-05-15 14:00]` 在 5 月 13 日 → "⏰ 2 天后"
  - `[once: 2026-05-10 09:00]` 在 5 月 13 日 → "⏰ 已过 3 天" (orange)
  - `[deadline: 2026-05-15 18:00]` 与 once 同款渲染
  - hover 显完整 target toLocaleString
  - 折叠 / 展开 cat 不影响数字（页面 render 时实时计算）

## 不在本轮范围

- 没做 setInterval 每分钟刷新：用户切 panel / hover / 任意 re-render
  时数字自然更新；持续 setInterval 会让无人看的面板浪费 CPU
- 没用 toLocaleString relative format API（Intl.RelativeTimeFormat）：
  4 档手动 mapping 更可控；i18n 现仅中文
- 没在 hover preview tooltip 也显此字段：tooltip 已显 schedule chip
  对应信息，重复展示噪声
- 没让 past 任务自动加 "立即处理" 快捷按钮：仍要走既有"立即处理 (N)"
  入口；inline 加按钮挤 row
