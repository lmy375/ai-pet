# PanelSettings "↺ 重置默认" SOUL.md 按钮

## 背景

TODO 原条目说 "PanelPersona 加..."。实际上 PanelPersona 是只读视图
（companionship / persona_summary / mood trend），SOUL.md 编辑器在
PanelSettings 顶部。本 iter 把按钮放到正确位置（PanelSettings 的 SOUL.md
section），dev log 此条同步声明。

## 需求

用户改 SOUL.md 改坏了 / 想从零开始时没复位入口 —— 当前只能去文件
系统 `~/.config/pet/SOUL.md` 删文件。给一个 inline 按钮 + armed 二
次确认。

## 实现

### 后端

`src-tauri/src/commands/settings.rs`：

- 新 tauri command `reset_soul_to_default() -> Result<String, String>`：
  - 调既有 `default_soul()` 拿内置默认串
  - 写入 SOUL.md（与 save_soul 同路径 / 同错误处理）
  - 返回新写入内容让前端 sync state 不必再 fetch

`src-tauri/src/lib.rs`：注册命令。

### 前端

`src/components/panel/PanelSettings.tsx` SOUL section：

- 新 state `soulResetArmed` + `soulResetBusy` + `soulResetTimerRef`
- SOUL.md 标题行加一个 "↺ 重置默认" 按钮（与 h4 同行 flex justify-between）：
  - default state 灰边白底
  - armed state 红边红字 + bold（5s 内有效）
  - busy state slate gray
  - onClick 二次确认：首点 set armed + 5s revert timer；再点 invoke
    reset_soul_to_default → setSoul(fresh) → setMessage 反馈
  - tooltip 详细解释覆盖语义（不可恢复）

## 验证

- `cargo check`：clean
- `npx tsc --noEmit`：clean
- 行为：
  - 改 SOUL.md textarea → 想复位 → 点 "↺ 重置默认" → 红字 "再点确认 (5s)"
  - 5s 内再点 → 灰字 "重置中…" → textarea 显内置默认 → message "SOUL.md
    已重置为内置默认"
  - 5s 不点 → revert default 灰边白底，无副作用
  - reset 不动 config.yaml / memory / sessions —— 与既有"重置默认"
    （清 settings）独立

## 不在本轮范围

- 没做"重置到上次保存的版本"（undo）：需要 backend 维护 SOUL.md 版本
  历史；scope 翻倍
- 没让"重置默认"也 reload 整个 panel：当前只 sync textarea state；
  其它字段不变更
- 没在 PanelPersona 也加同款按钮：那是只读视图，没编辑入口；不该塞
  reset
