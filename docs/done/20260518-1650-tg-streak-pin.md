# TG `/streak_pin` 命令（iter #579）— attention 维度对偶 /streak

## Background

既有 `/streak` 量化「连续 done 天数」= 完成度 attention 信号。owner
还想 audit「关注度 attention」— 多久没钉过任务。本命令补 pin 维度
streak。

完成 audit family：
- /streak: 连续 done 天数 + 7d/30d 总数 (完成度)
- /streak_pin: 连续 pinned active 天数 (关注度) ← 本 iter

## Algorithm: `compute_pin_streak(dates, has_current_pinned, today)`

pure helper — caller 已扫 butler_history 提 [pinned] sighting dates +
当前 pinned count + 今日 date：

```rust
// 1. 当前 streak：walk 从 today 倒退
let mut streak = 0;
let mut cursor = today;
loop {
    let s = date_str(cursor);
    let hit = dates_with_sighting.contains(&s)
        || (cursor == today && has_current_pinned); // today fallback
    if !hit { break; }
    streak += 1;
    cursor -= 1 day;
}
// 2. max streak in retention：BTreeSet sort + walk consecutive
```

**今日 fallback**：current pinned 数 > 0 时今日强算 — 即使今日 history
无 [pinned] sighting（长 pinned task 不每天更新），仍能识别「我现在有
钉的活」。

## Key design decisions

- **今日 fallback by has_current_pinned**：长 pinned task 不每天 update
  → history 无每日 sighting；但 owner 心智里「我今天仍有 pinned task」
  = 1 天。fallback 修这个 false negative
- **历史最长 streak 也含 fallback today**：BTreeSet 含 today（若 fallback）
  让连续段 detection 一致
- **`[pinned]` sighting on date-only key**：取 ts 前 10 字（RFC3339
  `YYYY-MM-DD` ASCII safe）— 不 parse 完整 datetime
- **chat-scoped views for current pinned**：read_tg_chat_task_views 已
  filter chat origin；与 /pinned / /pinned_drop_7d 同 scope。但 history
  sighting **不 chat-scope**（butler_history 是 global log，无 chat
  origin 字段）。pragmatic tradeoff：跨 chat sighting 会污染单 chat
  audit；owner 通常单 chat 用 pet，可接受
- **8 unit tests**：parser + 5 compute cases (empty / today fallback /
  consecutive / break on gap / max historical chain) + 2 formatter
  (empty fallback + full summary)
