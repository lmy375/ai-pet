# PanelTasks 右键菜单 "🔗 复制为 ref" 入口

## 需求

iter #178/#182/#188 把 `「title」` 全角直角引号包裹的 ref token 升级
成 chat 与 task panel 的导航语法（hover 显 status / 双击跳卡片）。
但用户想把任务引用粘到 chat / 外部 markdown / IM 时，要自己用 IME
打全角 `「」`。给一个 "复制为 ref" 入口，直接出可粘贴的 token。

## 实现

`src/components/panel/PanelTasks.tsx`：

任务卡右键菜单本来就分组了 copy 系列动作（📋 复制标题 / 📑 复制为
Markdown）。在两者之间加 🔗 复制为 ref：

- onClick：`navigator.clipboard.writeText(`「${m.title}」`)`
- 复制成功 / 失败均用既有 `setBulkResultMsg` 通道显 3s toast
- 关掉 ctx 菜单（与同级 entries 同流程）
- 文案 "🔗 复制为 ref（「title」）" —— 明确告诉用户复制出来的是什么形态

放在 "复制标题" 与 "复制为 Markdown" 之间是按"信息量从小到大"自然排
序：title 字符串 → ref token（含全角引号包装）→ 完整 markdown 段落。

## 验证

- `npx tsc --noEmit`：clean
- 行为：
  - 右键任意任务卡 → 菜单显 🔗 复制为 ref（「title」）
  - 点击 → 剪贴板 = `「整理 Downloads」`
  - 3s toast "已复制 ref：「整理 Downloads」"
  - 粘到 PanelChat → 自动渲为 dotted underline + status hover tooltip + 双击跳回该任务
  - 粘到外部（VSCode / Slack / Notion）→ 显纯文本 `「整理 Downloads」`

## 不在本轮范围

- 没把 entry 放到任务卡 action 行（inline button）：右键菜单已经聚类
  3 个 copy 动作；inline button 行已经挤（rename / status / fire 反馈等），
  再塞一个增加视觉噪声且使用频率边际
- 没做"复制全部选中任务的 ref（拼一段）"：批量场景少；当前 bulk 操作
  在 select 多选后已有"导出 markdown"，足够覆盖
- 没改 ⌘K picker 插入语义：picker 仍走 「title」 + 尾空格的"插入到 chat
  textarea 光标位"路径，与外部复制是两个动作
