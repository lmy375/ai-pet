# Pet 窗口拖图顶部 toast

## 需求

上一轮加了 window-level 拖图监听（拖到 Live2D / ChatMini 区也推到 ChatPanel
pendingImages）。但 pendingImages 缩略图条贴在窗口底部 ChatPanel 内，用户从
Finder 拖到顶部 Live2D 时眼睛在上面，看不到底部反馈——感觉"是不是没接住"。
顶部加一条 1.8s 自清的小 toast 让"图收到"立刻可见。

## 实现

`src/App.tsx`：

- 新 state `imageDropToast: number`（最近一次 drop 的张数，0 = 不显）
- 在已有 window drop handler 内 FileReader 全部完成时，除了 dispatch
  CustomEvent，还 `setImageDropToast(urls.length)` 并用 ts 时戳比对让旧
  timer 不提前清新 toast：
  ```ts
  const myTs = Date.now();
  setTimeout(() => setImageDropToast((cur) => Date.now() - myTs >= 1700 ? 0 : cur), 1800);
  ```
- pet 窗口顶部 absolute 渲染 toast：
  - 仅 `imageDropToast > 0` 时渲
  - 位置 top:6 + left:50% + translateX(-50%)，z 60
  - accent 底白字 + box-shadow，pointerEvents none 不挡 Live2D 鼠标交互
  - 文案 "📎 收到 N 张图 · 输入文本后回车发送"
  - 内嵌 `<style>` 注入 keyframes `pet-drop-toast-fade`：0→15→80→100%
    fade-in/hold/fade-out + slight y 漂移，1.8s 单跑

## 验证

- `npx tsc --noEmit` clean
- 行为：
  - 从 Finder 拖一张 png 到 Live2D 区域 → 顶部立刻浮"📎 收到 1 张图 ·
    输入文本后回车发送"，1.8s fade out
  - 连续拖 3 张 → toast 数字更新到 3
  - 拖到 ChatPanel 输入框（inner onDrop）→ App.tsx defaultPrevented 守门
    → toast 不显（ChatPanel 自己已经显缩略图条作为反馈）
  - toast 期间能继续拖动窗口 / 点 Live2D（pointerEvents none）
  - 窗口收起到 tab 状态时 toast 不渲染（在 `!hidden && (...)` 内）

## 不在本轮范围

- 没把 pendingImages 缩略图条挪到 ChatPanel 之上（TODO 文字描述的直译）：
  那需要把 pendingImages state 提到 App.tsx 共享，重构面太大；toast 已能
  解决"看不到反馈"主诉
- 没区分"拖入区域"高亮：拖中（onDragOver）让窗口边缘加蓝边的视觉效果
  可作单独需求
