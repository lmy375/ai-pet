# PanelDebug "🧹 重置 stash" 按钮

## 需求

调 prompt / 测 "冷启" 行为时，PanelDebug 的"看上次 prompt" modal、
"上次 manual fire" audit 行、recent turns 导航等都基于进程内静态
stash。旧 stash 在迭代调整 system soul 后会让用户看到陈旧数据混着
新行为，影响判断。给一个一键清 in-process stash 按钮（不动磁盘，
butler_history / memory / sessions / decision log 都不受影响）。

## 实现

### 后端

`src-tauri/src/proactive/telemetry.rs`：

- 新 tauri command `reset_proactive_stash() -> Result<(), String>`
- 清空 8 个 mutex 内的 Option / Vec / VecDeque：
  - `LAST_PROACTIVE_PROMPT` / `LAST_PROACTIVE_REPLY` / `LAST_PROACTIVE_TIMESTAMP`
  - `LAST_FEEDBACK_RECORDED_FOR`
  - `FORCED_TASK_FOCUS` / `LAST_MANUAL_FIRE`
  - `LAST_PROACTIVE_TOOLS` / `LAST_PROACTIVE_TURNS`
- 保留 `LAST_SEEN_BUTLER_DONE_TITLES`（dedup 状态用，清掉会让下一轮重
  复点名已完成任务，对正常运行有副作用）；调试场景一般不参与

`src-tauri/src/lib.rs`：注册 `proactive::reset_proactive_stash`。

### 前端

`src/components/panel/PanelDebug.tsx`：

- 新 state `resetStashArmed` + `resetStashBusy`，与 triggerArmed 同二次
  确认模式（首点变红 + 3s 内再点真触发，超时自动 revert）
- 在 DevTools 按钮后插 "🧹 重置 stash" 按钮：
  - default 灰底 slate-500 / armed 红边红字 / busy slate gray
  - 文案三态："🧹 重置 stash" / "再点确认 (3s)" / "清理中…"
- onClick 调 invoke + 完成后镜像清前端 cache：
  - `setLastManualFire(null)`（让 audit 行消失）
  - `setRecentTurns([])` + `setTurnIndex(0)`（让"看上次 prompt" modal
    重置）
- 反馈：`proactiveStatus` 复用现有 toast 通道 4s 自清

## 验证

- `cargo check`：clean
- `npx tsc --noEmit`：clean
- 行为：
  - 触发过一次 fire → audit 行浮 / "看上次 prompt" 内有 turn
  - 点 🧹 重置 stash → 按钮变红 + "再点确认 (3s)"
  - 3s 内再点 → 按钮变灰 "清理中…" → 完成 "✓ in-process stash 已清空"
  - audit 行消失 / modal 内 recentTurns 为空
  - 3s 不点 → 按钮 revert，无副作用
  - 重置后再 fire 一次 → 新 stash 出现，行为同冷启
  - butler_history.log / memory / sessions / decision log 都未动（验证：
    PanelMemory butler_tasks 历史段仍有原条目）

## 不在本轮范围

- 没清 `LAST_SEEN_BUTLER_DONE_TITLES`：那是"刚完成点名" dedup 状态，
  清掉会让宠物下一轮主动开口又点名一遍已完成任务，副作用太大
- 没做"分段 reset"（仅清 manual fire / 仅清 turns）：单按钮 + 一次清
  够用；分粒度需要 dropdown 选择，UI 重
- 没清 decision_log（磁盘）/ feedback_history.log（磁盘）：这两个是
  持久数据，不在 stash 范围
- 没做 reset 后自动跳转到 modal / 自动 fire 一次：用户清完通常自己
  会点"立即开口"测；自动化让"清空 → 重测"流程更顺但 scope 扩
