# TG `/here_status` 命令（iter #598）— transient_note 状态查看

## Background

iter #591-#595 加 here-* 注入 5 命令 + clear。owner 注入后想确认「pet
看到的 context 是什么 / 还剩多久失效」缺直接入口。

本 iter 补 /here_status — 列当前 transient_note 内容 + 到期 + 剩余分钟。

## Output

active:
```
📝 当前 transient_note（到 HH:MM 失效，剩 N 分钟）
· <text>
```

empty:
```
📝 当前无 transient_note。
试 /here_pin / /here_idle / /here_top_cat / /here_recent_done / /transient 注入。
```

expired (clock skew or refresh lag):
```
📝 当前 transient_note（到 HH:MM 失效，已过期）
· <text>
```

## Key design decisions

- **专用 status 命令而非 /aware 复用**：/aware 显完整 pet 感知（含
  transient + tasks + mood + 等），信息量大。/here_status 仅看 transient
  一行 — 精准查询
- **`remaining_minutes` 单独算 + format**：`(until - now).num_minutes()`
  负值 → 「已过期」label；正值 → 「剩 N 分钟」
- **`Option<i64>` for remaining_minutes**：None = 无 transient（不应
  到这分支）；Some(neg) = expired 边界（一般 get_transient_note 已过滤
  掉过期但 defensive）
- **教学指 5 个 here-* 注入入口**：empty fallback 提示全部注入路径让
  owner 不必记 family list
- **4 unit tests**：parser + empty fallback + active（含 remaining）
  + expired（负 minutes → 「已过期」label）
