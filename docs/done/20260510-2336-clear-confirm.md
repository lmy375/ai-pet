# /clear 二次确认

## 需求

`/clear` 直接清空当前 session 的 items / messages（虽然不删 session 文件，但 chat 内容确实没了），误触代价不小 —— 比如用户要敲 `/clean` 但 typo 成 `/clear`，整段对话上下文消失。需要一道便宜的确认门槛。

## 实现

走 PanelChat 内已经成熟的 5s armed pattern（同 `pendingDeleteId` for session）：

```ts
const [clearArmed, setClearArmed] = useState(false);
const clearArmTimerRef = useRef<number | null>(null);
```

`case "clear"`：
- 未 armed：setClearArmed(true) + 5s 自动 disarm，pushLocalAssistantNote `⚠️ 再敲一次 /clear 确认清空当前会话（5 秒内）。` 然后 break，不真清
- armed：clearTimeout、disarm，走原有清空逻辑（messagesRef = [sys]、setItems([])、save_session）

`executeSlash` 的 useCallback deps 加 `clearArmed`，保证第二次敲时拿到最新 state。

## 验证

- `npx tsc --noEmit` clean
- 行为：
  1. 敲 `/clear` → 出现 ⚠️ 提示，session 内容仍在
  2. 5 秒内再敲 `/clear` → session 真清空
  3. 等 5 秒后再敲 → 又是第一次"再敲一次"，需要再连按

## 不在本轮范围

- 没在 UI 上加可视化倒计时（5 秒小数字 / 进度条），降本：assistant 提示已经写明"5 秒内"，加倒计时引入额外渲染和层叠 z-index 调试；如果用户反馈光看一行文字不够再扩
- /clear 之外的破坏性 slash（目前没有）若以后加，按需复用同 pattern
