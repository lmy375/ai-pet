# 字号 / 行高 / 字重 / 字距 token scale（UI 美化 迭代 26）

## 背景

grep 统计：panel 体系内 `fontSize:` 出现 700+ 次，主流分布 10/11/12/13/14 px。多个 panel 的 sectionTitle 都用 13/13.5 之间分别拍脑袋；陈旧代码混用 `"11px"` 字符串和 `11` 数字。没有命名 scale，未来想"整体上抬一档"得改 700+ 处。

## 改动

### 新建 `src/text.ts`

四组常量：
- `text`：xs(10) / sm(11) / base(12) / md(13) / lg(14) / xl(16) / hd(28)
- `lineHeight`：tight(1.2) / snug(1.4) / base(1.55) / relaxed(1.65)
- `fontWeight`：normal(400) / medium(500) / semibold(600)
- `letterSpacing`：normal(0) / wide(0.1) / wider(0.2) / widest(0.3)

设计依据：grep 实际分布 + Tailwind 风格命名（团队心智模型迁移友好）。

不强制迁移：旧代码继续用数字字面量；新代码 / 重要 surface 用命名 token。

### 应用到 3 个共享组件作为 seed

- `SectionTitle.tsx`：fontSize 13.5→`text.md`，weight 600→`fontWeight.semibold`，letterSpacing 0.2→`letterSpacing.wider`，lineHeight 1.2→`lineHeight.tight`；副标 11→`text.sm` + 0.1→`letterSpacing.wide`
- `EmptyState.tsx`：title fontSize 13/12→`text.md/text.base`，weight 500→`medium`，hint 12/11→`text.base/text.sm`，lineHeight 1.5→`lineHeight.base`
- `LoadingState.tsx`：同 EmptyState 思路，三档 size 用 `inline ? text.sm : compact ? text.base : text.md`

视觉上 13.5→13 是 0.5px 下调，hard to notice。其它替换都是 1:1。

## 不做

- 不批量迁移 700+ inline 字面量。本轮仅奠基 + seed。
- 不引入 typography helper 组件（`<Heading size="md">` 等）—— React Native 风，与 inline style 习惯有摩擦。caller 用 `text.xxx` 当属性值即可。

## 验收

- `npx tsc --noEmit` 通过。
- 三个共享组件视觉与之前肉眼一致（除 SectionTitle 字号 13.5→13 微差异）。
- 后续要批量调字号节奏只改 `src/text.ts`。
