# TG `/pin_grow_7d` 命令（iter #562）— /pinned_drop_7d 正向对偶

## Background

`/pinned_drop_7d`（iter #560）audit「近 7 天放下了啥」反向关注度。
正向缺：「近 7 天新关注了啥 / 觉得啥变重要了」— priority 管理是否
健康循环的另一半信号。

## Detection (best-effort, mirrored from drop_7d)

```
candidates = title ∈ current_pinned_views      (live state)
           ∧ first [pinned] sighting in history ≥ now-7d
             (即：history retention 内首次能看到 [pinned] 是 7d 内 —
              7d 之前的 history 行不含 [pinned]，本 7d 内开始含 →
              视为「在 7d 窗口内被首次 pin」)
```

vs /pinned_drop_7d 用 `last [pinned] sighting ts` + 当前 unpinned，本
命令用 `first [pinned] sighting ts` + 当前 pinned。一行代码差（max vs
min），语义完全反向。

### Handler

复制 PinnedDrop7d 路径 + 改 max → min + filter 改为 first_ts ≥ cutoff：

```rust
// 取 first（min）ts per title — 与 drop 取 max 反向
first_pin_ts.entry(title.to_string())
    .and_modify(|prev| if ts_local < *prev { *prev = ts_local; })
    .or_insert(ts_local);

// 候选 filter：first_pin_ts ≥ cutoff
let with_dt = first_pin_ts.into_iter()
    .filter(|(_, ts)| *ts >= cutoff)
    .collect();
```

## Key design decisions

- **`first` 而非 `last` ts**：关键反差点。`first ≥ cutoff` = 「这条
  task 在 7d 之前 history 内没含 [pinned]，7d 内首次出现」= 新 pin。
  vs `last` 是「最近的 [pinned] sighting」（适合 drop 视角）
- **不取 history 内含 [pinned] 行全部 ts，仅取首次**：避免「这条
  task 在 7d 内多次 update 又 update（pin 状态没变）」造成的多次
  sighting 混淆 — 我们只关心「首次 sighting」这一个事件锚点
- **best-effort caveats**：与 /pinned_drop_7d 同 — snippet 80 字截断
  漏 marker、history retention 局限。但有额外一个：history retention
  之前已 pinned 的 task 也会被列（因为 retention 之前不可见，retention
  内"首次"sighting 在 7d 内 → 误判新 pin）。help-detail 内已说明
- **3 unit tests**：parser + empty fallback + multi-row（first 标签
  phrasing 验证）
