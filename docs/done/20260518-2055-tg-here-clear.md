# TG `/here_clear` 命令（iter #593）— transient 注入命令撤回对偶

## Background

iter #591 / #592 加了 /here_pin / /here_idle — 把 task 清单注 transient
_note。owner 用错了 / 想换主题想撤回 — 当前只能等 60min 自动到期，
或调 `/transient _ 0` 间接清（参数怪异，不直观）。

本 iter 加直接撤回入口 /here_clear — pipe owner 心智：「注入 → 撤回」
对偶完整。

## Changes

6-surface 同步：

1. Enum `HereClear`（紧贴 `HereIdle`）
2. name / parser / no-args title chain / registry en+zh / help-detail /
   help-table / 三份 drift lists
3. 新 pure `format_here_clear_reply(prior_text: Option<&str>)` formatter
4. Handler fetch get_transient_note → 调 set_transient_note("", 0) →
   formatter (含 50-char preview)
5. /help_table system 家族 + family detail map 同步加入

## Output

有 prior transient:
```
🧹 已清 transient_note
· 之前内容：📌 当前 pin context：「写周报」「整理 Downloads」...
```

空（无 transient）:
```
🧹 已无 transient_note — 无需清。
相关：/here_pin / /here_idle / /transient 看注入入口。
```

## Key design decisions

- **复用 `set_transient_note("", 0)` clear sentinel**：proactive.rs doc
  明示「Empty text or 0 minutes clears」— 不需新 backend 命令
- **prior preview 50-char utf8-safe 截**：让 owner 看到「清的是什么」
  — 与 `/here_pin` 注入时显 titles 一样：撤回操作可见。长 text 截
  + `…` 后缀
- **`Option<&str>` 参数**：caller 传 None / Some("") 都走「无 transient」
  分支；显式区分让 caller 灵活
- **空时不当 error**：友好提示「无需清」+ 教学指注入入口。比 "ERROR
  no transient" 更 owner-friendly
- **5 unit tests**：parser + 4 formatter case（None / 空 string /
  short prior 无截断 / long prior 50-char 截断 + …）
