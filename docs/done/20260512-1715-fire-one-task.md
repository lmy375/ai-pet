# PanelMemory butler_tasks 单条 "▶️ 现在跑一次" 按钮

## 需求

butler_tasks 段 header 已有 "立即处理 (N)" 按钮（overdue > 0 时浮），
调 `trigger_proactive_turn` —— LLM 在 butler_tasks_hint 全队列里自
己挑。但用户想指定特定 task 跑（"我现在就要这条 X 推进，不要让你挑"），
没有 per-item 入口。补 per-item 按钮 + 后端 forced-focus 注入。

## 实现

### 后端

`src-tauri/src/proactive/telemetry.rs`：

- 新增 `pub static FORCED_TASK_FOCUS: Mutex<Option<String>>`：panel 命令
  塞入目标 title，下一次 `run_proactive_turn` 起跑时 take（消费式 read）
- 用 take 而非 clone：保证某一次自然 tick / 普通 manual fire 不会继承
  上一次塞的意图

`src-tauri/src/proactive.rs`：

- `run_proactive_turn` 在算完 `butler_tasks_hint` 后检查 FORCED_TASK_FOCUS：
  - 命中 → 把 "⚡ 用户在面板上点击了「X」的『现在跑一次』按钮，请把本
    轮重心放在这条任务上：要么推进它、要么写一句进展、要么标 done /
    error..." 拼到 hint 最前
  - 不命中 → 原 hint 不变
- 新命令 `trigger_proactive_turn_for_task(title)`：
  - 校验 title 非空
  - 写入 FORCED_TASK_FOCUS
  - 用 RAII `ClearOnDrop` 兜底：panic / 早 return 路径都会清空 static，
    防止 stale title 漏到下一次自然 tick
  - 调既有 `trigger_proactive_turn(app)` —— 后端流程同源（速率统计 /
    feedback / decision log 都正常）

`src-tauri/src/lib.rs`：注册新命令 `trigger_proactive_turn_for_task`。

### 前端

`src/components/panel/PanelMemory.tsx`：

- 新 state `fireOneArmedTitle: string | null` + `fireOneArmedTimer` ref
- `handleFireOneTask(title)`：与现 `handleFireProactive` 同结构
  - 复用全局 `firingProactive` in-flight flag —— 多条任务并存时按一
    条 LLM 会处理那条，其他按钮 disabled 防连点
  - 同款 3s armed 二次确认（与现 fireArmed / handleDelete 等模式一
    致）
- 在 butler_tasks 段的每条 item 行 action buttons 区加新按钮：
  - 仅 `catKey === "butler_tasks"` 浮（其它 category 没 schedule 概念）
  - 三态文案：默认 "▶️ 现在跑" / armed "再点确认 (3s)" / busy "处理中…"
  - armed 状态加红色描边 + bold 视觉，与全局 "立即处理" 一致
  - busy 但 not-armed 状态 opacity 0.5 + cursor:default + disabled，
    其它任务的按钮可见但禁用

## 验证

- `cargo check`：clean（5 warning 全 unrelated）
- `npx tsc --noEmit`：clean
- 行为：
  - 在 butler_tasks 某 item 行点 "▶️ 现在跑" → 变 "再点确认 (3s)"
  - 3s 内再点 → invoke trigger_proactive_turn_for_task → button 变
    "处理中…"，其它任务的 "▶️" 按钮变灰 disabled
  - LLM 回复完 → message 显结果，loadIndex / loadButlerHistory 刷新
  - 3s 不点 → armed 自动 revert，按钮回 "▶️ 现在跑"
  - 同时多条 armed 不存在（fireOneArmedTitle 单值 state）
  - 后端：LLM prompt 中 butler_tasks_hint 最前面多了一条 "⚡ 用户在
    面板上点击了「X」的『现在跑一次』..." 强势指令
  - 若 trigger_proactive_turn_for_task 路径中 panic / 早 return →
    ClearOnDrop 清空 FORCED_TASK_FOCUS，下一次自然 tick 不继承

## 不在本轮范围

- 没做"队列 / 心跳 / 完成"三段 hint 同时点名（only butler_tasks_hint）：
  LLM 看到强势指令已经会优先处理；其它 hint 仍存在让 LLM 把这条放进
  全局上下文判断
- 没做"指定 task 的 detail.md / history 单独 inject 进 prompt"：当前
  仅插一行 directive，让 LLM 自己根据已有的 butler_tasks_hint 找到 X
  的 schedule / topic
- 没做 forced focus 失效检测（如 X 不在队列）：prompt 文案里说"如该
  任务不在下面的队列里说明已被归档 / 重命名，回一句简短确认即可"，
  让 LLM 自己处理 edge case；不用后端再做一次 lookup
- 没做"批量跑选中 N 条"：单条已能覆盖最常见诉求；批量需要 UI 选择 +
  排队，单独 iter
- 没做 visual：每条 task 当前 status / 心跳标记不变 —— 按钮是 verb
  动作，与 task 状态显示是两个 axis
